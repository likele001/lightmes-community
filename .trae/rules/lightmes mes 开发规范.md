【项目名称】LightMes 轻量化MES管理系统（KeleMes重构版）
【核心约束 必须严格遵守】
1.  开发环境：Linux，运行环境：Windows/Linux通用，给客户提供一键启动脚本（run.bat/start.sh）
2.  技术栈：Python 3.10+ FastAPI + SQLAlchemy + MySQL 5.7+，禁止使用Docker、容器、虚拟机
3.  数据库：必须兼容MySQL 5.7+，不允许使用MySQL 8.0专属语法
4.  业务必须实现完整闭环：客户下单→订单→生产计划→派工→扫码报工→审核→计件工资→溯源→对账
5.  工价 = 产品型号 + 工序，工资 = 审核通过报工 × 工价
6.  权限严格隔离：管理员/班组长/员工/客户，数据只能看自己的
7.  报工必须支持扫码、上传图片/视频，分工自动生成二维码
8.  代码可直接运行，注释清晰，不允许简化核心业务
9.  **接口规范**: 所有 API 统一返回格式 `{code:200, msg:"", data:{}}`。
10.  **权限控制**: 必须实现严格的角色与数据权限隔离，员工只能查看本人数据，客户只能查看自有订单。
11.  **核心业务逻辑**:
    - 产品与产品型号必须为两个独立模块，型号关联产品。
    - 工价必须与「产品型号 + 工序」强关联。
    - 工资计算必须基于「审核通过的报工记录」。
    - 报工数据必须包含多媒体证据（图片/视频）上传功能。
12. **可运行性**: 所有代码必须可直接运行，注释清晰，不得生成无法运行的伪代码。
13. **沟通语言**: 如果有可能尽量用中文 我能懂的更快
