## 第一阶段：系统初期配置

### 1.1 用户管理

**📌 使用者**: 系统管理员 / 超级管理员

**🔗 操作路径**: 系统 → 用户管理

**✅ 操作步骤**:
1. 点击「+ 新建用户」按钮
2. 填写基本信息:
   - 用户名（必填）：建议格式 `employee_001`、`manager_01` 等
   - 密码（必填）：初始密码，建议设置强密码
   - 姓名（必填）
   - 部门（可选）：选择所属部门
   - 手机号（可选）：用于接收通知/工资条
   - 邮箱（可选）
3. 选择角色（必填）：从下拉菜单中选择一个或多个角色
4. 点击「保存」
5. 新建用户会收到系统通知邮件（如配置了邮件服务）

**⚠️ 注意事项**:
- 初始密码建议**由系统生成或强密码**，不要用简单密码如 `123456`
- 用户名**全局唯一**，一旦创建不可更改
- 分配角色时，**一个用户可以绑定多个角色**，系统会合并权限
- 删除用户前，**检查该用户是否有待处理的任务**
- 员工离职时，**建议禁用用户而非删除**，保留操作日志
- 第一次登录后，**强制用户修改密码**

**🔍 相关操作**:
- 编辑用户：点击用户行 → 修改 → 保存
- 禁用用户：点击用户行 → 禁用（禁用后无法登录）
- 重置密码：点击用户行 → 重置密码 → 发送临时密码

### 1.2 角色管理

**📌 使用者**: 系统管理员 / 超级管理员

**🔗 操作路径**: 系统 → 角色管理

**✅ 操作步骤**:
1. 查看内置角色（系统预设）:
   - 超级管理员：全部权限
   - 厂长：生产/财务/报表管理
   - 生产计划员：订单/排产/派工
   - 班组长：派工审核/报工审核
   - 操作员工：报工/查看任务
   - 客户：下单/查看进度
   - 财务：工资/对账/报表

2. 自定义新角色（如需要）:
   - 点击「+ 新建角色」
   - 输入角色名称（如 `质检主管`）
   - 在权限列表中**逐一勾选**需要的权限点
   - 点击「保存」

3. 编辑已有角色权限:
   - 点击角色名 → 修改权限 → 保存

**⚠️ 注意事项**:
- **最小权限原则**：只分配必要的权限，不要给所有权限
- 权限更新后，**当前登录用户需要重新登录**才能生效
- **删除角色前**，确认没有用户绑定该角色
- 内置的6个角色**建议不要修改**，可以创建新角色
- 权限分为**视图权限**（查看）和**操作权限**（增删改）

**权限点对照表**:
| 权限点 | 说明 | 适用角色 |
|------|------|--------|
| dashboard.view | 仪表盘查看 | 所有人 |
| user.manage | 用户管理 | 系统管理员 |
| role.manage | 角色管理 | 系统管理员 |
| product.manage | 产品管理 | 主数据管理员 |
| sku.manage | 型号管理 | 主数据管理员 |
| price.manage | 工价设置 | 财务/主数据管理员 |
| order.manage | 订单管理 | 生产计划员/销售 |
| plan.manage | 生产计划 | 生产计划员 |
| task.manage | 任务派工 | 生产主管/班组长 |
| report.audit | 报工审核 | 班组长/QC |
| salary.manage | 工资管理 | 财务/班组长 |
| trace.query | 溯源查询 | 质管/客户服务 |

### 1.3 部门管理

**📌 使用者**: 系统管理员 / HR

**🔗 操作路径**: 系统 → 部门管理

**✅ 操作步骤**:
1. 点击「+ 新建部门」
2. 填写部门信息:
   - 部门名称（必填）：如 `冲压车间`、`组装部`、`质检部`
   - 负责人（可选）：下拉选择部门主管
   - 描述（可选）
3. 点击「保存」

4. 建立部门层级（如需要）:
   - 选择部门 → 编辑 → 设置上级部门 → 保存

**⚠️ 注意事项**:
- **部门名称**应简洁清晰，建议不要超过20个字
- 负责人一般是**该部门的班组长或主管**
- 如果启用了**数据权限控制**，用户只能查看自己部门的数据
- 删除部门前，**确认该部门没有员工**

### 1.4 字典管理

**📌 使用者**: 系统管理员 / 主数据管理员

**🔗 操作路径**: 系统 → 字典管理

**✅ 操作步骤**:
1. 查看系统预置的字典分类:
   - 单位（个、米、吨等）
   - 颜色（红、蓝、黄等）
   - 工序分类（机加、热处理等）
   - 订单状态（草稿、已确认等）
   - 其他业务字典

2. 新增字典值:
   - 选择字典分类 → 点击「+ 新建」
   - 输入字典值代码（如 `unit_pcs`）
   - 输入字典值文本（如 `个`）
   - 设置排序值（决定显示顺序）
   - 点击「保存」

3. 编辑/删除字典值:
   - 点击对应行 → 编辑或删除

**⚠️ 注意事项**:
- **字典值代码一旦保存就不能改**，建议建立命名规范
- 删除字典值前，**检查是否已被数据引用**
- 字典修改后，**页面可能需要刷新**才能看到新值
- **不要删除系统预置的关键字典**（如订单状态、工序类型）
- 颜色字典可以设置**十六进制色值**，方便前端显示

## 第二阶段：主数据建立

### 2.1 产品管理

**📌 使用者**: 主数据管理员 / 生产部

**🔗 操作路径**: 主数据 → 产品

**✅ 操作步骤**:
1. 点击「+ 新建产品」
2. 填写基本信息:
   - 产品编码（必填、唯一）：如 `P001`、`PUMP-100` 等
   - 产品名称（必填）：如 `离心泵`、`减速机`
   - 分类（可选）：选择产品大类
   - 单位（可选）：个、套、米等
   - 描述（可选）：产品详细说明
   - 主图（可选）：上传产品照片

3. 点击「保存」

4. 批量维护产品:
   - 导入 Excel：下载模板 → 填充数据 → 上传

**⚠️ 注意事项**:
- 产品编码**全局唯一，建立编码规范**（如 P + 3位数字）
- **产品名称要简洁清晰**，方便员工快速识别
- 一个产品可以对应**多个型号**（SKU），如不同颜色/规格
- 产品建立后**可以在型号中关联**
- **删除产品前**要确认没有关联的订单/型号

**示例**:
```
产品编码: P001
产品名称: 3匹离心泵
分类: 泵类
单位: 台
描述: 铸铁离心泵，用于工业冷却水循环
```

### 2.2 产品型号（SKU）管理

**📌 使用者**: 主数据管理员 / 销售

**🔗 操作路径**: 主数据 → 型号

**✅ 操作步骤**:
1. 点击「+ 新建型号」
2. 填写基本信息:
   - 选择关联产品（必填）
   - 型号编码（必填、唯一）：如 `SKU-P001-RED-L` 
   - 型号名称（自动或手动）
   - 颜色（可选）：从字典中选择
   - 材料（可选）：铸铁、钢等
   - 规格（可选）：尺寸、重量等自定义字段
   - 备注（可选）：特殊说明
   - 多角度图片（可选）：上传正面/侧面/细节图

3. 点击「保存」

4. **批量导入型号**（推荐）:
   - 下载 Excel 模板
   - 填充产品、颜色、材料、规格等信息
   - 点击「导入 Excel」 → 选择文件 → 上传
   - 系统提示导入结果（成功数/失败数）

**⚠️ 注意事项**:
- 型号编码建议格式：`产品编码-颜色-尺寸` 如 `P001-RED-L`
- **一个产品对应多个型号很正常**（如产品有3种颜色、2种规格 = 6个型号）
- 颜色字典**应该提前维护好**，这样导入时能快速关联
- 批量导入时，**第一行是表头，不要删除**
- 导入失败时会显示错误行号，修改后**重新上传**
- **关闭不生产的型号**（设置状态为停用），而非删除

**示例**:
```
产品ID: 1 (离心泵)
型号编码: SKU-P001-RED-L
颜色: 红
材料: 铸铁
规格: L-100
```

### 2.3 工序管理

**📌 使用者**: 工程/主数据管理员

**🔗 操作路径**: 主数据 → 工序

**✅ 操作步骤**:
1. 点击「+ 新建工序」
2. 填写工序信息:
   - 工序编码（必填、唯一）：如 `P001`、`WL001`
   - 工序名称（必填）：如 `下料`、`冲压`、`焊接`、`喷漆`
   - 所属车间（可选）：如 `冲压车间`、`组装部`
   - 标准工时（可选）：单位分钟，如 5 分钟/件
   - 责任人（可选）：工序负责人

3. 点击「保存」

**⚠️ 注意事项**:
- 工序编码**建立统一规范**，如 `P + 3位数字`
- **工序名称不要太长**，控制在10个字以内
- 标准工时用于**计划排产估算产能**，要准确
- 工序流程应该是**独立的、可重复的、标准化的操作**
- 一般工序分为：下料 → 加工 → 组装 → 检验 → 包装 这样的流程

**示例工序体系**:
```
P001 - 下料
P002 - 冲压
P003 - 焊接
P004 - 钻孔
P005 - 组装
P006 - 喷漆
P007 - 检验
P008 - 包装
```

### 2.4 工艺路线管理

**📌 使用者**: 工程 / 主数据管理员

**🔗 操作路径**: 主数据 → 工艺路线

**✅ 操作步骤**:
1. 点击「+ 新建工艺路线」
2. 填写基本信息:
   - 选择产品（必填）
   - 工艺路线名称（必填）：如 `离心泵标准工艺`
   - 是否为默认路线：勾选表示该产品的默认工艺

3. 添加工序步骤:
   - 点击「+ 添加工序」
   - 选择工序（从下拉列表中选择）
   - 设置步骤顺序（1、2、3...）
   - 可选：备注说明
   - 点击「确认」

4. 调整工序顺序:
   - 拖拽工序行可调整顺序，或直接编辑序号

5. 点击「保存工艺路线」

**⚠️ 注意事项**:
- **一个产品可以有多条工艺路线**（如标准工艺、加急工艺、特殊定制工艺）
- **标记一条为默认工艺**，生成订单时会自动应用
- 工序顺序**非常关键**，影响到后续工单分解
- 修改已应用到生产计划的工艺路线，**只影响新订单**，已生成的工单不变
- 工艺路线**一旦应用到订单，就不能删除**（可以禁用）

**示例**:
```
产品: 离心泵 (P001)
工艺路线: 标准工艺 (默认)
├─ 顺序1: P001 下料
├─ 顺序2: P002 冲压
├─ 顺序3: P003 焊接
├─ 顺序4: P007 检验
└─ 顺序5: P008 包装
```

### 2.5 物料管理

**📌 使用者**: 采购 / 仓库管理员

**🔗 操作路径**: 主数据 → 物料

**✅ 操作步骤**:
1. 点击「+ 新建物料」
2. 填写物料信息:
   - 物料编码（必填、唯一）：如 `M001`、`STL-100`
   - 物料名称（必填）：如 `铸铁`、`螺钉M8`
   - 规格型号（可选）：如 `直径100mm`
   - 单位（必填）：从字典中选择
   - 供应商（可选）：选择主要供应商
   - 最小库存量（可选）：库存告急提示值
   - 描述（可选）

3. 点击「保存」

**⚠️ 注意事项**:
- 物料编码**建立清晰规范**，便于采购和仓库识别
- **区分「产品」和「物料」**：
  - 产品 = 销售给客户的成品
  - 物料 = 用于生产产品的原材料/半成品
- 设置**合理的最小库存量**，系统会在库存低于此值时告警
- 供应商**可以在采购单时再选**，物料界面可选

### 2.6 BOM（物料清单）管理

**📌 使用者**: 工程 / 采购

**🔗 操作路径**: 主数据 → BOM

**✅ 操作步骤**:
1. 点击「+ 新建 BOM」
2. 填写基本信息:
   - 选择产品（必填）
   - 选择产品型号（可选）：可以是产品型号级别的 BOM
   - BOM 名称（必填）

3. 添加物料清单:
   - 点击「+ 添加物料」
   - 选择物料
   - 输入需用量（必填）：如生产1件产品需要多少物料
   - 输入单位（从字典选择）
   - 损耗率（可选）：如10%损耗
   - 点击「确认」

4. 完成后点击「保存 BOM」

**⚠️ 注意事项**:
- **BOM 用于生产排产时的物料需求计算**
- 损耗率不要填过高，建议 5%-15%
- BOM 可以在**产品级和型号级**分别建立
- 生成生产计划前，**确保 BOM 已完整配置**
- 物料清单确认无误后**锁定版本**，避免随意修改

**示例**:
```
产品: 离心泵 (P001)
BOM名称: 标准BOM

物料清单:
- 铸铁 (M001): 10 kg (损耗5%)
- 螺钉M8 (M002): 20 个
- 密封圈 (M003): 2 个
- 轴承 (M004): 1 个
```

## 第三阶段：工价设置

### 3.1 工序工价设置

**📌 使用者**: 财务 / 运营主管

**🔗 操作路径**: 主数据 → 工价

**✅ 操作步骤**:

**方式一：按型号批量设置**
1. 选择产品型号（必填）
2. 在弹出的「工价管理」界面中，查看该型号的所有工序
3. 逐一输入每个工序的**计件单价**（单位：元/件）
   - 工序名称 | 标准工时 | **单价(¥)** ← 输入
4. 点击「保存」

**方式二：手动逐个设置**
1. 点击「+ 新建工价」
2. 选择产品型号（必填）
3. 选择工序（必填）
4. 输入单价（必填）：如 5 元/件
5. 设置生效日期（可选）：默认立即生效
6. 点击「保存」

**⚠️ 注意事项**:
- **工价 = 产品型号 + 工序 + 单价**，这个组合唯一
- 一个型号可能有**多个工序**，每个工序工价**可能不同**
- 工价修改后，**只影响新订单**，已生成订单按当时的工价结算
- **历史工价要保留**，用于订单追溯和薪资审计
- 工价应该**定期评估**（如年度或季度），调整时要通知员工
- **单价填写要精确到小数点后两位**（如 5.50 元/件）

**示例**:
```
产品型号: SKU-P001-RED-L (离心泵-红-L)
├─ 工序1: 下料 - 0.50 ¥/件
├─ 工序2: 冲压 - 1.00 ¥/件
├─ 工序3: 焊接 - 2.00 ¥/件
├─ 工序4: 组装 - 1.50 ¥/件
├─ 工序5: 检验 - 0.50 ¥/件
└─ 工序6: 包装 - 0.30 ¥/件
  合计: 5.80 ¥/件
```

## 第四阶段：订单与生产

### 4.1 订单管理

**📌 使用者**: 销售 / 生产计划员

**🔗 操作路径**: 生产 → 订单

**✅ 操作步骤**:

**方式一：手动创建订单**
1. 点击「+ 新建订单」
2. 填写基本信息:
   - 选择客户（必填）：从客户库中选择
   - 订单编号（自动生成或手动输入）
   - 交期（必填）：格式 YYYY-MM-DD
   - 备注（可选）

3. 添加订单明细:
   - 点击「+ 添加产品」
   - 选择产品型号（必填）
   - 输入订单数量（必填）
   - 输入单价（可选，默认用产品价表）
   - 点击「确认」

4. 完成订单信息，点击「保存」（状态：草稿）

5. 订单确认:
   - 订单保存后状态为「草稿」
   - 点击「确认订单」→ 状态变为「已确认」
   - **确认后系统自动生成工单并分解工序任务**

**方式二：批量导入订单**
1. 下载 Excel 模板
2. 填充订单信息（客户、产品型号、数量、交期）
3. 点击「导入 Excel」
4. 选择文件 → 上传
5. 系统提示导入结果

**⚠️ 注意事项**:
- **订单号建议格式**：`2026050101` (日期+序号)，确保唯一
- **交期很关键**，影响生产排产和工资计算
- 订单在「草稿」状态可以修改和删除
- **订单确认后不能再修改或删除**，只能作废
- 如要作废已确认订单，点击「作废」，系统会记录原因
- 产品型号**必须已经建立**，订单才能创建
- **订单数量、交期、备注**一定要准确，关系到后续生产

**订单生命周期**:
```
草稿 → 已确认 → 生产中 → 已完成 → 已发货 / 已取消
```

## 第六阶段：报工与审核

### 6.1 报工（员工端 - 手机/H5）

**📌 使用者**: 操作员工

**🔗 操作路径**: H5 移动端 → 报工

**✅ 操作步骤**:
1. 打开 H5 移动端首页
2. 点击「我的任务」
3. 查看当前待报工任务列表

4. 报工操作（两种方式）:

**方式一：扫码报工**（推荐）
- 使用手机摄像头扫描任务二维码
- 系统自动加载该任务信息
- 跳转到报工表单

**方式二：手动选择**
- 从列表中点击任务
- 进入报工表单

5. 填写报工信息:
   - 合格数量（必填）：实际完成的合格产品数
   - 不良数量（可选）：出现问题的产品数
   - 完工照片（可选）：上传1-5张现场照片
   - 完工视频（可选）：上传实际工作过程视频
   - 备注（可选）：如遇到的问题

- 报工后**不能修改**，如有错误需要班组长驳回重新报
- **交接班时要及时报工**，不要积压

### 6.2 报工审核（班组长/QC 后台）

**📌 使用者**: 班组长 / QC / 质检主管

**🔗 操作路径**: 生产 → 报工审核

**✅ 操作步骤**:
1. 查看待审核报工列表（状态：待初审）
2. 点击报工记录查看详情:
   - 操作员工名称和ID
   - 任务编号/产品型号
   - 合格数/不良数
   - 上传的照片/视频

3. **初审（班组长）**:
   - 选择「批准」或「驳回」
   - 如批准：点击「批准」
   - 如驳回：点击「驳回」→ 填写**驳回原因**（必填）→ 确认
   - 驳回后报工回到员工，员工需要**重新报工**

4. **终审（QC/质检）**:
   - 查看已初审通过的报工（状态：待终审）
   - 重点抽查：
     - 查看上传的照片/视频
     - 对比合格数/不良数的合理性
     - 检查不良率是否超标
   - 操作同初审：批准或驳回

5. **批准后自动入账**:
   - 终审批准的报工自动成为有效数据
   - 该笔数据自动计入**该员工的工资表**

**⚠️ 注意事项**:
- **初审很重要**，班组长要确保数据真实性
- **驳回一定要填原因**，便于员工改正
- **抽查照片/视频**，确保报工数据不虚报
- 不良率超标（如>5%）的报工**需要重点关注**
- **一旦终审批准就无法修改**，所以审核要仔细
- 如果发现虚报，可以**批量驳回**该员工的报工

**报工状态流转**:
```
已报工 → 待初审(班组长) → 待终审(QC) → 已通过(自动入账) / 已驳回(重新报工)
```

## 第七阶段：工资结算

### 7.1 工资自动计算

**📌 使用者**: 财务 / 班组长

**🔗 操作路径**: 生产 → 工资管理

**✅ 操作步骤**:
1. 进入工资管理首页
2. 选择统计月份（如 2026-05）
3. 系统自动计算该月所有员工的工资:
   - 数据来源：该月**审核通过的报工**
   - 计算公式：
     ```
     基础工资 = Σ(报工数量 × 对应工序工价)
     补贴 = 全勤奖 + 岗位津贴 + ...
     扣款 = 迟到扣款 + 其他扣款 + ...
     月工资 = 基础工资 + 补贴 - 扣款
     ```

4. 查看员工工资明细:
   - 点击员工名称
   - 显示该员工的报工明细和计算过程
   - 可见每个工序的报工数量和金额

5. 调整补贴/扣款:
   - 如需要添加全勤奖：输入金额 → 保存
   - 如需要减少迟到扣款：输入金额 → 保存

6. 工资确认和导出:
   - 所有工资计算完成后，点击「确认」
   - 状态变为「已确认」，员工可查看工资条
   - 点击「导出 Excel」生成工资表
   - 导出的 Excel 包含所有明细，可用于财务审核

**⚠️ 注意事项**:
- **工资完全基于审核通过的报工**，虚报报工会影响工资
- **补贴和扣款要准确**，建议**提前制定统一标准**
- 工资数据确认前**可以修改补贴/扣款**，确认后就锁定
- **确认后员工就能看到工资条**
- 如果有员工申请**工资复审**，提供明细表进行解释
- **保留原始计算数据**，以备审计

**⚠️ 注意事项**:

- **定期分析毛利**，识别低毛利订单或产品
- 如毛利率低，分析原因：
  - 工价定得太低？
  - 物料损耗过大？
  - 产能低？
- **用毛利分析指导定价策略**

## 第十阶段：溯源与报表

### 10.1 溯源查询

**📌 使用者**: 质管 / 客户服务 / 工程

**🔗 操作路径**: 生产 → 溯源查询

**✅ 操作步骤**:
1. 输入或扫描成品二维码/条码:
   - 手动输入成品编号
   - 或使用手机扫描二维码

2. 系统反查该产品的全生命周期信息:
   - **订单信息**：订单号、客户、交期
   - **工单信息**：工单号、产品型号、数量
   - **工序追踪**：每个工序的：
     - 操作员工名称
     - 操作时间
     - 完成数量
     - 报工照片/视频
   - **质检记录**：QC 审核意见、不良率
   - **物料批次**：使用的物料来源和批次号
   - **设备记录**：使用的设备和工序参数

3. 生成溯源报告:
   - 点击「生成报告」
   - 生成 PDF 溯源证书
   - 可发送给客户作为质量证明

**⚠️ 注意事项**:
- **溯源信息完整性很重要**，确保所有工序都有记录
- **报工时必须上传照片/视频**，否则无法溯源
- **一旦生产完成，溯源信息就固定了**，无法修改
- **溯源用于：质量问题追责、客户投诉处理、产品召回**

**📌 使用者**: 设备管理员 / 生产主管

**🔗 操作路径**: 生产 → 设备管理

### 11.2 日常点检

**📌 使用者**: 班组长 / 设备员

**🔗 操作路径**: 生产 → 设备管理 → 列表「点检」

**✅ 操作步骤**:
1. 在对应设备行点击 **点检**
2. 系统默认登记「日检 / 合格」
3. 点检数据供 AI 设备健康分析参考

**⚠️ 注意事项**:
- **点检**适合每日快速确认；**保养**用于按计划执行的维护并留档
- 异常设备应改为「维修中」状态并补充保养说明

---

### 11.3 保养计划（预防性维护）

**📌 使用者**: 设备管理员

**🔗 操作路径**: 生产 → 设备管理 →「保养」→ 保养计划

**✅ 操作步骤**:
1. 点击设备行的 **保养**，打开保养管理抽屉
2. 点击 **新增计划**
3. 配置：
   - **计划类型**：日检 / 周检 / 月检
   - **周期(天)**：如月检填 `30`
   - **下次日期**：首次计划执行日
   - **负责人**：可选
   - **检查项**：如「润滑；紧固；清洁」
4. 保存计划

**⚠️ 注意事项**:
- 同一设备可配置多条计划（如周检 + 月检）
- 周期天数用于 **登记保养后自动推算下次维护日期**

---

### 11.4 登记保养记录

**📌 使用者**: 设备员 / 班组长

**🔗 操作路径**: 保养抽屉 → 计划行「执行」或「登记保养」

**✅ 操作步骤**:
1. **从计划执行**：点击计划行的 **执行** → 选择结果（合格/不合格/部分完成）→ 填写说明 → 提交
2. **直接登记**：点击 **登记保养** → 可选关联计划 → 填写结果与说明 → 提交
3. 提交后列表 **上次维护** 更新为当天；若关联计划且有周期，**下次维护** 自动顺延

**⚠️ 注意事项**:
- 不合格保养应填写说明，便于追溯
- 删除保养计划不会删除历史保养记录

## 📋 系统常见问题与解决方案

### 问题1：订单确认后发现产品型号错了，怎么办？
**解决**:
- 已确认的订单**不能直接修改**
- 操作：作废该订单 → 新建正确的订单
- 系统会记录作废原因和时间

### 问题2：报工提交后发现数字填错了，怎么办？
**解决**:
- 已提交的报工**不能自己修改**
- 联系班组长，请班组长「驳回」该报工
- 员工重新报工（可以修改数字）

### 问题3：工资计算似乎有误，怎么复查？
**解决**:
- 进入「工资管理」
- 点击员工名称查看详细计算过程
- 每笔报工的数量和金额都有记录
- 如确实有误，通知财务进行调整

### 问题4：物料库存不足，排产计划能推进吗？
**解决**:
- 排产前系统会提示「物料不齐套」
- 两个选择：
  - A. 延期排产（等物料到货）
  - B. 紧急采购（加急订货）
- **不建议在物料不足的情况下排产**，会导致停工

### 问题5：订单已交期，还没完成，怎么办？
**解决**:
- 首页仪表盘会高亮显示超期订单
- 进度看板也会标红
- 立即协调：
  - 加派人力
  - 加班处理
  - 或与客户沟通延期

### 问题6：员工对工资有异议，怎么处理？
**解决**:
- 直接看工资管理里的明细
- 每笔报工的工序和金额都有记录
- 如有虚报报工，可以显示报工照片/视频对质
- 确实有算错的，财务进行调整

### 问题7：如何导出打印工资表给财务审核？
**解决**:
- 进入「工资管理」
- 点击「导出 Excel」
- 生成的 Excel 包含所有员工的完整工资明细
- 可以打印或发送给财务

### 问题8：产品的工艺路线要改，已生成的订单受影响吗？
**解决**:
- **已确认的订单按原工艺路线执行**，不受影响
- 只有**新确认的订单**才用新工艺路线
- 建议新旧工艺路线**同时维护一段时间**，再删除旧的

## 🎯 最佳实践建议

### 1. 数据准确性最重要
- 产品/型号/工价/BOM 一定要准确
- 报工数据关系到工资，员工很关注
- 定期审计数据，发现异常及时处理

### 2. 建立规范和标准
- 产品编码规范
- 订单号规范
- 字典值规范
- 这样便于数据查询和维护

### 3. 权限最小化原则
- 员工只能看自己的任务和工资
- 班组长能管理自己部门的工作
- 财务独立管理工资和对账
- 系统管理员谨慎分配权限

### 4. 定期备份和复盘
- 定期导出重要数据（工资表、订单表）
- 月度或季度复盘各项指标
- 识别问题和改进机会

### 5. 员工培训很关键
- 不同角色需要培训不同功能
- 班组长要理解报工审核的重要性
- 员工要知道如何正确报工
- 财务要理解工资计算逻辑
