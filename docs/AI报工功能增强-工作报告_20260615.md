## 一、总体概览

| 序号 | 功能 | 后端 | 前端 | 依赖 |

| 6 | 换班交接摘要 | `scenes.shift_handover_summary()` + API | HomePage 摘要卡片 | `_run_scene()` |

## 三、核心设计决策

### 3.1 AI 自动审核（风险评分模型）

```
评分维度 (总分 100):
  1. 历史良率（近 30 天）   25分  不良率越低分越低
  2. 历史驳回率（近 30 天） 20分  被驳回越多分越高
  3. 入职时长               10分  新员工分略高
  4. 报工数量合理性          15分  超出剩余分高
  5. 照片证据完整            10分  无照片分高
  6. 不良率突增              20分  异常突增分高

阈值: <20 → low → 自动通过至 leader_approved
      20~59 → medium → 正常审核
      ≥60 → high → 强制人工 + 提醒
```

### 3.2 智能报工建议评分

```
评分维度:
  1. 上次报工时间（30分）  越久越优先
  2. 剩余数量（25分）      剩余多优先
  3. 订单交期（30分）      <7天加急
  4. 工序顺序（0~5分）    前道优先
  5. 任务状态（5分）       dispatched > in_progress
```

### 3.3 语音报工流程

```
用户按"录音"按钮 → [Web Speech API] → 语音→文字
  → [POST /ai/report/voice-parse] → LLM 结构化解析
  → 自动填入 good_qty / bad_qty / remark / defect_keywords
```
