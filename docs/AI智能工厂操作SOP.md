# LightMes AI 智能工厂操作 SOP

> **总手册已合并至 [LightMes生产自动化与智能中心操作手册.md](./LightMes生产自动化与智能中心操作手册.md)**。下文保留 AI 专项短流程。  
> 适用：管理端 + 员工 H5。排产/派工 **自动化开关** 见手册第四章；AI **只建议、不自动改库**（QC 终审默认人工）。

## 一、上线检查（Phase A）

1. **数据库**：`cd backend && alembic upgrade head`（当前 head 含 `0042_report_unit_prescreen`）
2. **后端**：重启 Uvicorn / Gunicorn
3. **Celery**：重启 worker 与 beat（预警定时任务 `ai.alerts.scan`）
4. **前端**：管理端 `npm run build`，整包覆盖 `dist` 并强刷浏览器
5. **健康检查**：`GET /api/health` → `build` 含 `ai-factory` 标记

## 二、平台 AI 配置

1. 平台运营 → **AI 模型** → 打开总开关
2. 新增网关（Base URL + API Key，OpenAI 兼容）
3. 网关下新增模型并设 **默认模型**；报工识图需另设 **Vision 模型**（勾选 is_vision）
4. 网关 **测试** 通过后再开放租户功能

## 三、订单 → 计划 → 派工 正确顺序

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 订单 **审核通过** | 状态 `confirmed`，尚未生成工单 |
| 2 | 新建 **生产计划** 并设置日期 | 状态 `planned` |
| 3 | **齐套检查** | 缺料可「允许缺料」下发 |
| 4 | **确认下发** | 生成工单与工序任务，计划 `in_progress` |
| 5 | **自动/手动派工** | 任务分配到员工 |

**常见误区**

- 「订单已审核」≠「计划已下发」—— 列表「已下发」以计划 `in_progress` 为准
- 「智能排产 → 采纳并执行」会尝试：排日期 → 自动下发 → 自动派工；缺料时需勾选允许缺料或先手动下发

## 四、智能排产（LLM + OR-Tools）

1. **生产计划** 列表 → **智能排产**
2. 查看 **LLM 文字建议** 与 **OR-Tools 约束方案** 对比
3. 选择方案后 **采纳并执行** 或仅手动调整日期后 **确认下发**

## 五、自动派工（技能 + 车间）

1. **系统 → 技能标签**：维护技能，给用户绑定技能
2. **基础数据 → 工序**：编辑工序，绑定 **所需技能**
3. **确认下发** 后 → **自动派工**
   - 仅 **员工** 角色（可选含班组长）
   - 按工序所需技能过滤；车间名与部门名匹配时优先本车间员工
   - 结合历史报工熟练度均衡负荷

## 六、报工与审核

| 角色 | 入口 | AI 能力 |
|------|------|---------|
| 员工 H5 | 逐件报工 | **AI 检查一下**（文字） |
| 审核员 | 件次报工审核 | **AI 批量摘要**、单条 **Vision 识图辅助** |

终审仍需上传质检附件；AI 结论仅供参考。

## 七、数据预警

- 首页 **AI 数据预警** 卡片；Celery 每日 8/12/16/20 点扫描
- **系统设置 → AI 预警阈值**（或 API `PUT /admin/ai/alert-settings`）可配置：
  - 待审报工积压阈值
  - 逾期订单提醒
  - 良率下降幅度
  - 未派工任务 backlog

## 八、故障排查

| 现象 | 处理 |
|------|------|
| AI 503 未配置 | 检查平台总开关、默认模型、网关 Key |
| 超时 120s | 重建前端；Nginx `proxy_read_timeout ≥ 120` |
| 派工无人 | 员工角色、工序技能、人员技能是否匹配 |
| BOM 500 | 执行 `0033_bom_scope` 迁移 |
| build 标记旧 | 后端未重启或未部署新代码 |

## 九、权限速查

| 权限 | 功能 |
|------|------|
| `ai.use` | 工厂助手、排产/交期 AI、手动跑预警 |
| `ai.alert.view` | 查看预警卡片 |
| `ai.report_assist` | H5 报工 AI 检查 |
| `plan.manage` | 智能排产、采纳执行 |
| `report.audit` | 审核 AI 摘要、Vision 辅助 |
