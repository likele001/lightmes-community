# LightMes 宝塔部署 — 常见错误排查

> 适用：宝塔面板 + Python 项目 + Nginx 静态前端 + Redis/Celery  
> 后端日志：`/www/wwwlogs/python/lightmes/error.log`  
> 站点 Nginx 日志：`/www/wwwlogs/admin.lightmes.user.023ent.net.log`（域名按实际替换）  
> 虚拟环境（本机标准）：`export VENV=/www/server/pyporject_evn/lightmes`（见 [宝塔部署.md](./宝塔部署.md) 二、2.2）

---

## 一、先确认三件事（30 秒）

在服务器终端执行：

```bash
export VENV=/www/server/pyporject_evn/lightmes

# 1. 后端是否活着
curl -s http://127.0.0.1:8000/api/health

# 2. 验证码依赖（Pillow）是否可导入
$VENV/bin/python -c "from PIL import Image; print('Pillow OK')"

# 3. Redis 是否在听 6379
/www/server/redis/src/redis-cli -h 127.0.0.1 ping
```

| 结果 | 含义 |
|------|------|
| health 返回 `"code":200` | 后端正常 |
| `Connection refused` / 无响应 | 后端未启动或 8000 无进程 → 见 **二、503** |
| `No module named 'PIL'` | 缺 Pillow → 见 **三** |
| Redis 非 `PONG` | Redis 未启动 → 见 **五** |

浏览器访问请用 **https://**，登录后 **Ctrl+F5** 强刷，避免缓存到失败状态。

## 二、503 Service Unavailable（API / 偶发静态资源）

### 现象

浏览器控制台例如：

```
GET https://admin.xxx.net/api/platform/public-config 503
GET https://admin.xxx.net/api/auth/captcha 503
AxiosError: Request failed with status code 503
```

有时连 `favicon.svg` 也显示 503（多发生在整站短暂不可用期间）。

### 原因

Nginx 将 `/api/` 反代到 `http://127.0.0.1:8000`。当 **uvicorn 未监听 8000** 时，网关返回 **502/503**，前端登录页、验证码、public-config 全部失败。

常见触发：

1. Python 项目 **启动失败**（见下文缺 Pillow）
2. 宝塔 **连续重启**，旧进程占 8000，新进程绑不上（`address already in use`）
3. 手动在 SSH 又起了一个 uvicorn，与面板进程冲突

### 日志特征

`/www/wwwlogs/python/lightmes/error.log` 中可见：

```
ModuleNotFoundError: No module named 'PIL'
ERROR: [Errno 98] error while attempting to bind on address ('127.0.0.1', 8000): address already in use
```

恢复后会有：

```
GET /api/platform/public-config HTTP/1.1" 200 OK
```

### 处理步骤

```bash
export VENV=/www/server/pyporject_evn/lightmes

# 1. 补依赖（验证码必需 Pillow）
$VENV/bin/pip install Pillow redis
$VENV/bin/pip install -r /www/wwwroot/lightmes/backend/requirements.txt

# 2. 避免多进程占 8000（宝塔里先【停止】Python 项目，再执行）
pkill -f "uvicorn app.main:app.*8000" || true
sleep 2

# 3. 宝塔 → Python 项目 → lightmes-backend → 【启动】（只点一次）

# 4. 自检
curl -s http://127.0.0.1:8000/api/health
curl -s http://127.0.0.1:8000/api/platform/public-config
```

浏览器用 **https://** 打开管理端，**Ctrl+F5** 强刷。

### 注意

- 站点 Nginx 只需反代 **`/api/`** → `127.0.0.1:8000`，不要把整站 `/` 都反代到后端。
- 静态资源（`/assets/*.js`）应由 `frontend-admin-pro/dist` 直接提供；若仅 API 503 而静态 200，基本可断定是后端问题。

## 三、ModuleNotFoundError: No module named 'PIL'

### 现象

`error.log` 启动栈：

```
File ".../app/services/captcha.py", line 12
    from PIL import Image, ImageDraw, ImageFont
ModuleNotFoundError: No module named 'PIL'
```

### 原因

登录验证码、图形验证码依赖 **Pillow**。宝塔 Python 虚拟环境未安装或未用正确 venv 执行 `pip install`。

### 处理

```bash
export VENV=/www/server/pyporject_evn/lightmes
$VENV/bin/pip install Pillow
# 或一次性装全
$VENV/bin/pip install -r /www/wwwroot/lightmes/backend/requirements.txt
```

然后在宝塔 **重启** Python 项目。

## 四、No module named 'sqlalchemy'（用错虚拟环境）

### 现象

在 `backend/.venv` 或系统 `python3` 下执行 `alembic` / `pytest` 报：

```
ModuleNotFoundError: No module named 'sqlalchemy'
```

### 原因

项目内 `backend/.venv` 可能与系统 Python 版本不一致（例如包装在 `python3.11` 目录但 `python3` 指向 3.12），**宝塔生产环境不使用该目录**。

### 处理

一律使用面板 **环境路径**（示例）：

```bash
export VENV=/www/server/pyporject_evn/lightmes
$VENV/bin/pip install -r /www/wwwroot/lightmes/backend/requirements.txt
$VENV/bin/alembic upgrade head
```

以宝塔 → Python 项目 → **环境路径** 显示为准，勿假设 `backend/.venv`。

## 五、Celery：Cannot connect to redis (Error 111)

### 现象

```
consumer: Cannot connect to redis://127.0.0.1:6379/0: Error 111 connecting to 127.0.0.1:6379. Connection refused.
```

### 原因

Redis 服务未启动。Celery、登录验证码（推荐）、生产自动化异步任务均依赖 Redis。

### 处理

```bash
# 宝塔 Redis 常见路径
/etc/init.d/redis status
LC_ALL=C LANG=C /etc/init.d/redis start
/www/server/redis/src/redis-cli ping   # 应返回 PONG
```

宝塔 → **软件商店** → **Redis** → **启动** + **开机启动**。

启动 Celery（使用正确 venv）：

```bash
bash /www/wwwroot/lightmes/scripts/start-celery.sh
# 或见 宝塔部署.md 第八节手动命令
```

建议用 **进程守护** 常驻，不要只开在 SSH 终端里。

## 六、address already in use（8000 端口占用）

### 现象

```
ERROR: [Errno 98] error while attempting to bind on address ('127.0.0.1', 8000): address already in use
```

### 原因

已有 uvicorn 占用 8000，宝塔再次启动失败，期间可能出现 503。

### 处理

```bash
ps aux | grep "uvicorn.*8000"
# 宝塔 Python 项目 → 【停止】
pkill -f "uvicorn app.main:app.*8000"
sleep 2
# 再【启动】一次
```

## 七、前端静态资源 404（与 503 区别）

### 现象

```
Failed to load resource: .../assets/WorkOrdersPage-xxx.js 404
TypeError: Failed to fetch dynamically imported module
```

### 原因

`index.html` 与 `assets/` 下 JS **版本不一致**（只上传了部分 dist、或未重新 build）。

### 处理

```bash
cd /www/wwwroot/lightmes/frontend-admin-pro
npm install && npm run build
```

将 **整个** `dist` 目录部署到站点根目录，浏览器 **Ctrl+F5**。

| 错误码 | 含义 |
|--------|------|
| **503** | 网关/后端不可用（多为 Python 未起） |
| **404** | 文件不存在（多为 dist 未整包更新） |

## 八、alembic: command not found

### 现象

直接执行 `alembic upgrade head` 报命令找不到。

### 处理

```bash
export VENV=/www/server/pyporject_evn/lightmes
cd /www/wwwroot/lightmes/backend
$VENV/bin/alembic upgrade head
$VENV/bin/alembic current
```

## 十一、Python 3.14 + SQLAlchemy 启动失败（AssertionError）

### 现象

`/www/wwwlogs/python/lightmes/error.log` 中，uvicorn 在 **import 阶段** 就退出，末尾类似：

```
AssertionError: Class <class 'sqlalchemy.sql.elements.SQLCoreOperations'> directly inherits TypingOnly but has additional attributes {'__firstlineno__', '__static_attributes__'}.
```

宝塔 Python 项目显示 **未启动** 或反复重启失败；`curl http://127.0.0.1:8000/api/health` 无响应。

### 原因

宝塔虚拟环境使用 **Python 3.14** 时，若 `pip install` 装到了 **SQLAlchemy 2.0.0**（或其它 < 2.0.36 的旧版），与 Python 3.14 的 typing 机制不兼容，**应用无法 import**。

这与 MySQL、Nginx 无关，纯依赖版本问题。

### 处理

```bash
export VENV=/www/server/pyporject_evn/lightmes

# 1. 升级 SQLAlchemy（必须 ≥ 2.0.36，推荐直接装 requirements）
$VENV/bin/pip install -U 'SQLAlchemy>=2.0.36'
# 或
$VENV/bin/pip install -r /www/wwwroot/lightmes/backend/requirements.txt

# 2. 验证能 import
cd /www/wwwroot/lightmes/backend
$VENV/bin/python -c "from app.main import app; print('import ok')"

# 3. 宝塔 → Python 项目 → lightmes → 【启动】（若 8000 被占，先【停止】再启）

# 4. 自检
curl -s http://127.0.0.1:8000/api/health
$VENV/bin/pip show sqlalchemy | grep Version   # 应 ≥ 2.0.36
```

### 预防

- 仓库 `backend/requirements.txt` 已写 `SQLAlchemy>=2.0.36`，重装依赖时用 **`$VENV/bin/pip install -r requirements.txt`**，不要单独 pin 旧版。
- 若仍频繁失败，可在宝塔新建 Python 项目时选 **Python 3.12**（与 SQLAlchemy 兼容面更广），环境路径需重新创建并迁移。

## 十二、同机多 Python 项目：8000 / 8008 端口冲突

### 现象

```
ERROR: [Errno 98] error while attempting to bind on address ('0.0.0.0', 8008): address already in use
```

或 lightmes **8000** 同样报 `address already in use`，但宝塔显示项目 **未启动**。

### 原因

一台机器上多个 Python 项目 **不能共用同一端口**。常见情况：

| 项目（宝塔名） | 端口 | 目录 |
|----------------|------|------|
| **lightmes** | **8000** | `/www/wwwroot/lightmes/backend` |
| **backend**（或其它 AI 项目） | **8008** | `/www/wwwroot/openai/backend` 等 |

8008 若已有 **遗留 uvicorn 进程**（之前手动或异常退出留下），再在宝塔点「启动」会失败并狂刷日志。

### 处理（以启动 8008 的 backend 为例）

```bash
# 1. 看谁占着端口
ss -tlnp | grep -E ':8000|:8008'

# 2. 清 8008 遗留进程（把 PID 换成上一步看到的）
kill <占用8008的PID>
sleep 1
ss -tlnp | grep 8008    # 应无输出

# 3. 宝塔 → Python 项目 → backend → 【启动】

# 4. 自检
curl -s http://127.0.0.1:8008/api/health
curl -s http://127.0.0.1:8000/api/health
```

**lightmes 占 8000 同理**：宝塔先【停止】→ `pkill -f "uvicorn app.main:app.*8000"` → 再【启动】一次。

### 注意

- **LightMes 前端 Nginx 反代只指向 8000**，与 8008 的其它项目无关。
- 不要把两个项目的端口配成一样；目录名带 `openai` 只是文件夹名，宝塔里项目名可以是 `backend`。

## 十三、`.env` 被 source 时报 Backend: command not found

### 现象

`error.log` 开头一行：

```
/www/wwwroot/lightmes/backend/.env: line 1: Backend: command not found
```

### 原因

宝塔启动脚本会 `source .env`。若第一行写成：

```ini
APP_NAME=LightMes Backend
```

shell 会把 `Backend` 当成命令执行。

### 处理

改为带引号（仓库 `.env.example` 建议同步）：

```ini
APP_NAME="LightMes Backend"
```

保存后宝塔 **重启** lightmes 即可。

## 十四、日志位置速查

| 类型 | 路径 |
|------|------|
| Python 项目（uvicorn） | `/www/wwwlogs/python/lightmes/error.log` |
| 管理端 Nginx 访问 | `/www/wwwlogs/admin.lightmes.user.023ent.net.log` |
| 管理端 Nginx 错误 | `/www/wwwlogs/admin.lightmes.user.023ent.net.error.log` |
| Redis | `/www/server/redis/redis.log` |

实时查看 Python 错误：

```bash
tail -f /www/wwwlogs/python/lightmes/error.log
```

## 十五、部署后推荐自检清单

```bash
export VENV=/www/server/pyporject_evn/lightmes
cd /www/wwwroot/lightmes/backend

$VENV/bin/pip install -r requirements.txt
$VENV/bin/pip show sqlalchemy | grep Version    # Python 3.14 时应 ≥ 2.0.36
$VENV/bin/python -c "from app.main import app; print('import ok')"
$VENV/bin/alembic upgrade head
curl -s http://127.0.0.1:8000/api/health | grep -o '"build":"[^"]*"'
/www/server/redis/src/redis-cli ping
```

- [ ] 宝塔 Python 项目 **lightmes 运行中**，仅 1 个 uvicorn 监听 **8000**
- [ ] `$VENV/bin/pip show sqlalchemy` 版本 **≥ 2.0.36**（Python 3.14 必查）
- [ ] Redis **PONG**，Celery worker + beat 已守护  
- [ ] admin-pro / h5 已 `npm run build` 并整包发布 dist  
- [ ] 浏览器 **https** 打开，public-config / captcha 返回 200
