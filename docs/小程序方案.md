# LightMes 微信小程序方案（uni-app v2）

> 工程：`lightmes-miniapp/` · 技术栈：uni-app + Vue3 + TS + Pinia  
> 范围：完整员工端 + PC 租户管理端全功能（不含客户自助端、不含 SaaS 平台运营端）

## 1. 架构

```
主包
├── shared/login、bind-account、role-select
└── tabs/tab-0..4（自定义 TabBar，员工/管理双模式切换）

分包 pages-employee（8 页）
└── 报工/任务/工资/消息等（对齐 frontend-h5）

分包 pages-admin（56 页）
└── 对齐 frontend-admin-pro 全部租户模块
```

## 2. 权限与安全

| 层级 | 实现 |
|------|------|
| 登录 | `POST /miniapp/auth/login` + `GET /auth/me` |
| 员工 API | `/h5/*`，角色 `employee` / `leader` |
| 管理 API | `/admin/*`、`/dashboard/*`，请求头 `X-LightMes-Portal: admin` |
| 前端守卫 | `usePermission()` + `permissions[]` 控制菜单与页面 |
| 双角色 | `leader` 等可在「员工模式 / 管理模式」间切换 |

权限码定义：`src/constants/permissions.ts`（与 `backend/app/core/seed.py` 一致）。

## 3. 功能对照

### 员工端（100% 对齐 H5）

| 页面 | 路径 |
|------|------|
| 首页/任务/报工/打卡/我的 | Tab `tab-0`..`tab-4` |
| 扫码报工 | `pages-employee/report/scan` |
| 逐件报工 | `pages-employee/report/unit` |
| 工资/工资条签名 | `pages-employee/salary/*` |

### 管理端（对齐 PC）

系统、主数据、生产、CRM、计划、仓储、设备、采购、财务、报表、看板等 56 个列表/详情页，采用 **MListLayout** 移动列表范式。

## 4. 部署

1. `npm install && npm run build:mp-weixin`
2. 微信开发者工具打开 `dist/build/mp-weixin`
3. 配置合法域名、`VITE_API_BASE_URL`、小程序 AppID
4. 后端 `WX_MINIAPP_*` 环境变量
