> 适用：PC 管理端（`frontend-admin-pro`）  
> 权限：`equipment.manage`  
> 数据库迁移：`0043_equipment_maintenance`（依赖 `3b617c3fe6e7`）

## 一、功能列表

| 功能 | 说明 | 入口 / API |
|------|------|------------|
| **设备档案** | 编码、名称、型号、车间、状态（正常/维修中/已退役） | 生产 → 设备管理；`GET/POST /admin/equipment` |
| **编辑设备** | 修改名称、型号、车间、状态 | 列表「编辑」；`PUT /admin/equipment/{id}` |
| **日常点检** | 快速登记日检结果（默认合格） | 列表「点检」；`POST /admin/equipment/{id}/check` |
| **保养计划** | 日/周/月计划、检查项、周期天数、负责人、下次日期 | 列表「保养」抽屉；`GET/POST/PUT/DELETE /admin/equipment/maintenance-plans` |
| **保养记录** | 登记保养执行结果（合格/不合格/部分完成） | 保养抽屉「登记保养」或计划行「执行」；`GET/POST /admin/equipment/maintenance-logs` |
| **维护日期联动** | 登记保养后自动更新设备「上次维护」「下次维护」 | 关联计划且填写周期天数时，按周期推算下次日期 |
| **AI 设备健康** | 基于点检频次给出维护建议（规则版） | 智能中心 → 深度 AI；`GET /admin/ai/deep/equipment-health` |

## 三、操作教程

### 3.1 新增设备

1. 进入 **生产 → 设备管理**
2. 点击 **新增设备**
3. 填写名称（必填）、型号、车间；编码可留空自动生成
4. 保存后在列表中可见

### 3.2 日常点检

1. 在设备行点击 **点检**
2. 系统默认登记「日检 / 合格」
3. 点检记录可在后续版本中于详情中查看（当前 API：`GET /admin/equipment/{id}/checks`）

### 3.3 配置保养计划（预防性维护）

1. 点击设备行的 **保养**，打开保养管理抽屉
2. 在 **保养计划** 区域点击 **新增计划**
3. 填写：
   - **计划类型**：日检 / 周检 / 月检
   - **周期(天)**：如月检填 `30`
   - **下次日期**：计划首次执行日期
   - **负责人**：可选，从系统用户中选择
   - **检查项**：如「润滑；紧固；清洁滤网」
4. 保存后计划出现在列表中

### 3.4 执行保养并登记

**方式 A — 从计划执行**

1. 在计划行点击 **执行**
2. 选择 **保养结果**（合格 / 不合格 / 部分完成）
3. 填写说明（可选）→ **提交**

**方式 B — 直接登记**

1. 在 **保养记录** 区域点击 **登记保养**
2. 可选关联计划 → 填写结果与说明 → **提交**

**系统自动行为：**

- 更新设备 **上次维护** 为当天
- 若关联计划且设置了 **周期(天)**，则 **下次维护** 与计划 **下次日期** 同步为「今天 + 周期」

### 3.5 编辑 / 删除计划

- **编辑**：计划行 → 编辑 → 修改类型、周期、检查项等
- **删除**：计划行 → 删除（需确认）；历史保养记录保留，`plan_id` 可为空

## 四、API 速查

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/admin/equipment` | 设备列表 |
| POST | `/admin/equipment` | 新增设备 |
| PUT | `/admin/equipment/{id}` | 更新设备 |
| POST | `/admin/equipment/{id}/check` | 点检 |
| GET | `/admin/equipment/{id}/checks` | 点检历史 |
| GET | `/admin/equipment/maintenance-plans` | 保养计划列表（`?equipment_id=`） |
| POST | `/admin/equipment/maintenance-plans` | 新建计划 |
| PUT | `/admin/equipment/maintenance-plans/{id}` | 更新计划 |
| DELETE | `/admin/equipment/maintenance-plans/{id}` | 删除计划 |
| GET | `/admin/equipment/maintenance-logs` | 保养记录列表 |
| POST | `/admin/equipment/maintenance-logs` | 登记保养 |

请求体示例（新建计划）：

```json
{
  "equipment_id": 1,
  "plan_type": "monthly",
  "interval_days": 30,
  "next_date": "2026-06-01",
  "check_items": "润滑;紧固",
  "responsible_user_id": 2,
  "remark": "每月一号"
}
```

## 五、常见问题

| 现象 | 原因 | 处理 |
|------|------|------|
| 保养按钮无数据 / 404 | 后端未重启或未跑迁移 | `alembic upgrade head` + 重启 uvicorn |
| 登记保养后日期未变 | 未关联计划或未填周期 | 执行时选计划，或计划里设置 `interval_days` |
| 点检与保养区别 | 点检=日常快速打卡；保养=计划性维护+记录 | 按 SOP 分开使用 |
| Celery 任务不执行 | worker/beat 未启动 | `scripts/start-celery.sh`，见 [宝塔部署.md](./宝塔部署.md) 第八节 |
