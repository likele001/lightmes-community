# LightMes 钉钉消息推送部署指南

> 与飞书、企微并列的第三 IM 通道。配置入口：**系统管理 → 钉钉消息推送** 或 **统一消息中心**。

---

## 1. 开放平台配置

1. 登录 [钉钉开放平台](https://open.dingtalk.com/) → 创建**企业内部应用**
2. 记录 **CorpId、AppKey、AppSecret、AgentId**
3. 权限管理开通：通讯录只读、工作通知、根据手机号获取 userid
4. OAuth 回调地址（须 HTTPS 公网可访问）：

   ```
   https://<API域名>/api/dingtalk/oauth/callback
   ```

5. 各通知群添加**自定义机器人**，复制 Webhook（若开启加签，同时记录 Secret）

---

## 2. LightMes 管理端配置

路径：**系统管理 → 钉钉消息推送**（`/system/dingtalk-notify`）

| 项 | 说明 |
|----|------|
| 启用 | 总开关 |
| AppKey / AppSecret / AgentId | 应用凭证 |
| 群组 Webhook | 生产群 / 管理群 / 全厂群 |
| 报工卡片审核 | 开启后班组长钉钉工作通知带「初审通过/驳回」按钮 |
| API 外网地址 | 与 `PUBLIC_BASE_URL` 一致，用于 OAuth 与卡片审核链接 |

**统一消息中心**可同时维护三通道群组：`channels.dingtalk.webhook_url` + `webhook_secret`。

---

## 3. 人员绑定

| 方式 | 操作 |
|------|------|
| OAuth | H5 个人中心 → 绑定钉钉；或管理端「管理员 OAuth 绑定」 |
| 手机号匹配 | 钉钉推送页 → 按手机号批量匹配 userid（需员工资料手机号与钉钉一致） |
| 手工 | 管理端人员绑定表直接填写 userid |

---

## 4. 报工卡片审核（ActionCard）

钉钉按钮为 **URL 跳转**（非飞书式服务端推送）：

- 按钮指向 `GET /api/dingtalk/card-action?token=<JWT>`
- 服务端校验后执行报工初审/驳回，返回 HTML 结果页

联调：提交报工 → 班组长钉钉收到工作通知 → 点击按钮 → 页面显示操作结果。

---

## 5. 工厂日报

事件码 `brief.daily` 自动走统一 dispatcher，推送到已启用的**管理群 / 全厂群**钉钉 Webhook。  
详见 [工厂日报飞书企微推送配置指南.md](./工厂日报飞书企微推送配置指南.md)（已含钉钉说明）。

---

## 6. 运维

| 现象 | 处理 |
|------|------|
| 推送日志 `pending` | 检查 Celery worker 与 Redis |
| 工作通知失败 | 检查 AgentId、应用可见范围、userid 是否绑定 |
| Webhook 失败 | 检查 URL、加签 Secret、机器人是否被移出群 |
| 连续失败告警 | 消息中心 → 告警接收人 |

Celery 任务：`dingtalk.send_message`、`dingtalk.flush_deferred`（每 5 分钟）。
