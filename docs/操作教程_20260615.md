## 1. 报表导出

> **价值**：原报表只能在线查看，导出后可在 Excel 中二次加工、做月报、邮件转发。  
> **支持异步**：大数据量时后台生成，**几分钟后到「导出中心」下载**。

### 1.1 产量报表

- **使用者**：厂长 / 生产计划员 / 财务
- **操作路径**：Admin-Pro → 报表 → 产量报表 → 右上角「导出」

#### 操作步骤

1. 进入「产量报表」页面。
2. 顶部选择统计区间（**支持快捷区间**：今日 / 本周 / 本月 / 本季 / 自定义）。
3. 可选筛选：产品型号、车间、工序、操作员工。
4. 点击右上角「**导出**」按钮，弹出「异步导出」对话框。
5. 选「**Excel (.xlsx)**」或「**CSV**」格式 → 点击「**开始导出**」。
6. 系统提示「**导出任务已创建**」，可在「**导出中心**」（侧边栏：报表 → 导出中心）查看进度。
7. 状态变为「**已完成**」时，点击「**下载**」即可。

#### 注意事项

- 导出任务默认保留 **7 天**，到期自动清理，建议及时下载。
- 大数据量（> 10 万行）请选择 CSV 格式，更稳定、打开更快。
- 「导出中心」支持按「**报表类型**」「**创建时间**」「**状态**」筛选。
- 如果状态一直是「**处理中**」超过 30 分钟，刷新页面；仍异常请查看「附录 A Q1」。

#### 相关操作

- **查看历史**：导出中心 → 「我的导出」tab。
- **批量下载**：勾选多条「已完成」任务 → 「**打包下载**」。
- **删除任务**：选中后「**删除**」可立即释放存储。

---

### 1.2 良率报表

- **使用者**：厂长 / 质检主管 / 生产经理
- **操作路径**：Admin-Pro → 报表 → 良率报表 → 「导出」

#### 操作步骤

1. 进入「良率报表」页面。
2. 选择时间区间、车间、产品型号。
3. 页面默认显示「**合格率、不良率、各缺陷码占比**」图表。
4. 点击「**导出**」 → 选 Excel 格式 → 「开始导出」。
5. 导出 Excel 中除图表数据外，**额外包含「缺陷 TOP10」明细**（缺陷码、次数、占比）。

#### 注意事项

- 良率 = 合格数 / 总报工数；不良率 = 不良数 / 总报工数。
- 导出的「缺陷 TOP10」可作为 **质量改善专题** 的输入材料。
- 同期对比：勾选「**对比上期**」会在导出的 Excel 中多出对比列。

#### 相关操作

- **钻取分析**：点击某缺陷码 → 跳到「**缺陷分析**」详细页。
- **自定义列**：右上角「**列设置**」可勾选需要导出的字段。

---

### 1.3 库存报表

- **使用者**：仓库管理员 / 财务 / 采购
- **操作路径**：Admin-Pro → 仓库 → 库存 → 「导出」

#### 操作步骤

1. 进入「库存」页面。
2. 顶部筛选仓库、物料分类、库存状态（在库 / 安全库存以下 / 积压）。
3. 点击「**导出**」 → 选择「**当前快照**」或「**带流水**」。
   - **当前快照**：仅含当前库存数量。
   - **带流水**：除数量外，**包含最近 30 天的出入库流水**（适合月结对账）。
4. 导出任务到「**导出中心**」下载。

#### 注意事项

- 库存金额字段**需在「系统设置 → 计价方式」开启**后才显示。
- 「**安全库存**」以下的物料导出时会**标红**，便于重点跟进。
- **物料编码 / 批次号** 在导出的 Excel 中保留前导 0，不要被自动转成科学计数法。

#### 相关操作

- **设置安全库存**：物料详情 → 「库存参数」tab。
- **打印盘点表**：勾选多条 → 「**打印盘点表**」（A4 格式，可勾选空白列用于手填）。

### 1.4 对账单报表

- **使用者**：财务 / 销售客服
- **操作路径**：Admin-Pro → 财务 → 对账单 → 「导出」

#### 操作步骤

1. 进入「对账单」页面。
2. 选择客户、对账周期（如「2026-05」）。
3. 系统自动汇总该周期内所有已交付订单的金额、税额、已收 / 未收。
4. 点击「**导出**」 → 选择格式：
   - **对账单 Excel**：标准对账模板（含订单明细 + 汇总 + 客户信息）。
   - **对账单 PDF**：已加盖电子章，可直接发送给客户。
5. 任务完成后到「**导出中心**」下载。

#### 注意事项

- 导出 PDF 对账单需要**先在「系统设置 → 电子签章」上传公司印章**。
- 已确认的对账单导出的 PDF **带「已确认」水印**，与草稿状态可一眼区分。
- 客户邮箱发送：导出完成后，点「**邮件发送**」可直接发到客户档案里的邮箱。

#### 相关操作

- **批量导出**：勾选多份对账单 → 「**批量导出**」可一次性打 ZIP 包。
- **对接客户 H5**：对账单同步到客户的 H5 客户端 → 客户在线确认 / 异议。

## 2. 模具 / 工装管理

> **价值**：模具 / 工装是工厂的"重资产"，寿命到期前没做维保会引发批量质量事故。  
> 本模块把模具**全生命周期**（建档 → 使用 → 维保 → 报废）纳入系统。

### 2.1 模具档案

- **使用者**：设备管理员 / 生产主管
- **操作路径**：Admin-Pro → 生产 → 模具管理

#### 操作步骤

1. 进入「模具管理」页面。
2. 点击「**+ 新建模具**」。
3. 填写：
   - **模具编码**（必填，建议 `MOLD-` 前缀，如 `MOLD-001`）
   - **名称 / 类型**（注塑 / 冲压 / 压铸 / 工装夹具）
   - **关联产品**（可选，便于按产品查模具）
   - **设计寿命（次 / 件）**、**当前累计次数**（首次建档可填 0）
   - **存放位置**、**责任人**
4. 点击「**保存**」。

#### 列表字段说明

| 字段 | 说明 |
| --- | --- |
| 模具编码 | 唯一，扫描二维码即显示 |
| 寿命进度 | 进度条 + 百分比；**80% 黄色预警，100% 红色** |
| 累计次数 | 随员工报工自动累加 |
| 最近维保 | 上次维保日期；超期会标红 |
| 状态 | **正常 / 维保中 / 待报废** |

#### 注意事项

- 模具编码一旦保存**不可修改**。
- 关联产品后，**该产品的报工自动累计到模具**，无需手动登记。
- 导入模具：列表页「**导入**」下载模板 → 填充 → 上传。

#### 相关操作

- **扫码查看**：在 H5 「我的任务」或「生产工具」扫码，可查模具详情 / 维保历史。
- **打印模具卡**：详情页「**打印**」→ A6 模具卡（含二维码）。

---

### 2.2 寿命预警

- **使用者**：设备管理员 / 班组长
- **操作路径**：Admin-Pro → 生产 → 模具管理 → 顶部「**寿命预警**」

#### 触发规则

| 进度 | 颜色 | 系统动作 |
| --- | --- | --- |
| ≥ 80% | 🟡 黄色 | 列表页标黄 + IM 推送（设备管理员 / 班组长） |
| ≥ 95% | 🟠 橙色 | 推送升级到车间主任 |
| ≥ 100% | 🔴 红色 | **自动冻结关联产品的报工**，必须先维保才能继续 |

#### 接收预警

- 预警会通过**飞书 / 钉钉 / 企微**推送给：
  - 设备管理员（全员）
  - 关联车间的班组长
  - 厂长（红色预警时）
- 站内通知：右上角「**铃铛**」红点。

#### 处理方法

1. 收到预警 → 进入「模具管理」找到该模具。
2. 点击「**登记维保**」（详见 [2.3](#23-维保记录)）。
3. 维保完成后，**寿命进度清零**（或按实际剩余次数手动调整）。
4. 红色预警时，系统在**维保提交前冻结报工**；维保完成后自动解冻。

#### 注意事项

- 预警阈值可在「系统设置 → 模具参数」调整（默认 80/95/100）。
- 寿命进度支持**手动校正**（如更换了关键零件），但需填写校正原因。
- 误报处理：在「**预警记录**」页可标记为「已处理 / 误报」。

### 2.3 维保记录

- **使用者**：设备员 / 班组长
- **操作路径**：Admin-Pro → 生产 → 模具管理 → 详情 → 「**维保记录**」

#### 操作步骤

1. 找到目标模具 → 点击进入详情。
2. 「**维保记录**」tab → 点击「**+ 新建记录**」。
3. 选择**维保类型**：
   - **日常清洁**：最常用
   - **定期保养**：周 / 月保养
   - **故障维修**：非计划停机
   - **寿命校正**：更换零件后重置寿命
4. 填写**维保说明**、上传**现场照片**（最多 9 张）。
5. 「**保存**」 → 记录出现在列表 → 同步到「**预警记录**」。

#### 移动端登记（H5）

- 设备员在车间现场：H5 → 「**生产工具**」tab → 「**扫码登记维保**」→ 扫模具码 → 直接填写。
- 适合不方便开电脑的场景。

#### 注意事项

- **维修**类型必须勾选「**停机时长**」字段。
- 维保记录**不可删除**，如填错可「**作废**」（保留痕迹）。
- 寿命校正：填「**重置后次数**」，系统自动算出消耗量并写入历史。

#### 相关操作

- **导出维保台账**：列表「**导出**」按时间段生成 Excel。
- **关联备件**：维保类型选「故障维修」时，可勾选「**使用备件**」（自动扣减备件库存）。

## 3. 条码打印机直连

> **价值**：扫码出入库 / 报工时，**贴标即用**，无需人工二次打印。  
> LightMes 支持两种直连方式：**浏览器插件（Lodop）**和**ZPL 网络打印**。

### 3.1 浏览器 / Lodop 直连

- **使用者**：仓库 / 车间文员
- **操作路径**：Admin-Pro → 仓库 → 库存 / 工单 → 详情 → 「**打印标签**」

#### 首次使用：安装 Lodop 插件

1. 在 Admin-Pro 顶部「**系统管理**」→ 「**打印设置**」→ 「**下载 Lodop 控件**」。
2. 安装完成后**刷新页面**（Windows 可能需要重启浏览器）。
3. 点击「**测试打印**」→ 弹出窗口显示打印机列表 → 选打印机 → 打印测试页成功即可。

#### 操作步骤

1. 进入「库存 / 工单 / 产品型号」详情页。
2. 点击「**打印标签**」 → 选标签模板（如「产品码 50×30mm」）。
3. 选打印机 → 设置打印份数 → 「**打印**」。
4. Lodop 弹窗显示预览 → 点「**确定**」出标。

#### 注意事项

- **仅 Windows + Chrome / Edge / 360 浏览器**支持 Lodop。
- 标签模板可在「打印设置」按需添加（支持自定义尺寸、二维码位置、字段）。
- 打印失败排查：见「**附录 A Q2**」。

#### 相关操作

- **批量打印**：列表页勾选多条 → 「**批量打标**」。
- **按模板套打**：模板支持变量（产品名 / 编码 / 批次 / 数量），新打印时自动套用。

---

### 3.2 ZPL 网络打印

> **适用场景**：使用 Zebra / TSC 等工业条码机，机器固定 IP 接入车间网络。

- **使用者**：车间 IT / 仓库主管
- **操作路径**：Admin-Pro → 系统管理 → 打印设置 → 「**网络打印机**」

#### 添加打印机

1. 进入「网络打印机」页 → 「**+ 新增打印机**」。
2. 填写：
   - **打印机名称**（如 "车间 1 号 ZEBRA"）
   - **IP 地址**（如 192.168.1.100）
   - **端口**（默认 9100）
   - **DPI / 标签尺寸**（影响 ZPL 渲染）
3. 点击「**测试连接**」→ 看到「**测试页已发送**」即成功。
4. 保存。

#### 打印标签

- 「打印标签」时，**打印机下拉里选网络打印机**即可，系统自动生成 ZPL 指令并发送。
- 适合**多车间多打印机**的场景：不同车间 / 工序用不同打印机。

#### 注意事项

- 打印机与服务器**必须同网段**（或能互通）。
- **防火墙**：9100 端口需在打印机所在网络放行。
- 离线缓存：网络打印机暂时不通时，打印任务**自动入队**，恢复后**重试 3 次**。

#### 相关操作

- **打印机状态监控**：列表显示「**在线 / 离线 / 缺纸 / 忙**」。
- **打印历史**：每台打印机可查最近 100 条打印记录（标签内容 + 时间 + 结果）。

## 4. SPC 统计过程控制

> **价值**：在不良率"爆发"前，**提前发现工序偏移**，避免批量报废。  
> SPC（Statistical Process Control）通过控制图（Xbar-R、P、np、c、u）**实时监控**关键尺寸 / 缺陷率。

### 4.1 控制图管理

- **使用者**：质管部 / 工艺工程师
- **操作路径**：Admin-Pro → 生产 → SPC 控制图

#### 操作步骤

1. 进入「SPC 控制图」页面。
2. 点击「**+ 新建控制图**」。
3. 填写：
   - **名称**（如 "CNC 外径 Xbar-R"）
   - **图表类型**：
     - `Xbar-R`（计量型，最常用）：均值 + 极差
     - `P`（计件不良率）
     - `np`（不良数）
     - `c`（缺陷数）
     - `u`（单位缺陷数）
   - **关联工序**（如 OP-020 精车）
   - **关联产品**（可选）
   - **样本量**（默认 5）
   - **目标值 / 规格中心**（如 50.0 mm）
   - **UCL / LCL**（控制上下限，Xbar-R 图系统可**自动算**）
4. 点击「**保存**」。

#### 控制图类型选择建议

| 数据类型 | 推荐图表 | 典型场景 |
| --- | --- | --- |
| 尺寸 / 重量 / 长度 | Xbar-R | 机加工外径、注塑件重量 |
| 是否合格（计数） | P | 焊点合格率、装配一次合格率 |
| 缺陷数 | c | 表面缺陷数 |
| 每单位缺陷 | u | 铸件气孔数 / 100 件 |

#### 注意事项

- UCL / LCL **建议先点「自动计算」**，再根据经验微调。
- 控制图**仅监控**该工序；同一个工序的不同设备可建**多张**图。
- 控制图**可停用 / 启用**，停用后不再监控但保留历史。

#### 相关操作

- **复制控制图**：列表「**复制**」快速建同类型图（仅改名称 / 关联产品）。
- **导出控制图**：详情「**导出 PNG**」或「**导出 PDF**」。

### 4.2 样本录入与查看

- **使用者**：QC 质检员 / 班组长
- **操作路径**：Admin-Pro → 生产 → SPC 控制图 → 详情 → 「**样本录入**」

#### 手动录入

1. 在控制图详情页 → 「**样本录入**」tab。
2. 选择**测量批次**（每批 = 一次抽样的 N 个数据）。
3. 输入 N 个测量值（如 50.1, 50.2, 49.9, 50.0, 50.3）。
4. 点击「**保存**」 → 系统自动计算均值 / 极差 / 标准差。
5. 数据点落到控制图上，**越界点标红**。

#### 自动录入（推荐）

- 在 H5 「**报工**」时，员工上传**带尺寸水印的照片**，QC 用 AI 识别自动入库（**需启用 AI 测量服务**）。
- 与「[10. AI 缺陷分类](#10-ai-缺陷分类)」可联动。

#### 异常判定（判异规则）

控制图出现以下任一情况，**系统会红色提示 + IM 推送**：

| 规则 | 含义 |
| --- | --- |
| **1 点出界** | 任一数据点 > UCL 或 < LCL |
| **连续 9 点在中心线同侧** | 过程均值偏移 |
| **连续 6 点递增 / 递减** | 趋势性偏移 |
| **连续 2 点接近控制限（2σ 内）** | 即将失控 |

#### 注意事项

- **样本量必须 = 控制图设定的 N**，否则不计算。
- 录入时单位要与图设置一致（mm / cm / g 不可混用）。
- 录入错误可「**作废**」该批次（保留痕迹，不影响历史趋势）。

#### 相关操作

- **导出原始数据**：详情 → 「**导出 CSV**」。
- **打印控制图**：详情 → 「**打印**」 → A4 横版含图 + 判异记录。

### 4.3 Cpk 判异标准

> **Cpk（过程能力指数）** 是衡量"工序能否稳定满足规格"的指标，**越大越好**。

#### Cpk 判读表

| Cpk 值 | 评价 | 建议动作 |
| --- | --- | --- |
| **≥ 1.67** | 优（过剩） | 可考虑放宽公差以降低成本 |
| **1.33 ~ 1.67** | **充足**（行业标准） | 维持现状 |
| **1.00 ~ 1.33** | 尚可 | 关注 4M 变化（人 / 机 / 料 / 法） |
| **< 1.00** | 不足 | **必须停线整顿**，做 PFMEA |

#### Cpk 在哪里看

- 控制图详情页 → 「**能力分析**」面板。
- 系统**每月自动重算 Cpk**，并归档历史趋势图。

#### 注意事项

- Cpk 至少需要 **25 个子组（≥ 125 个数据点）** 才统计意义可靠。
- 数据非正态时，Cpk 仅供参考，可看「**Ppk**」（整体能力）。
- Cpk < 1 时，控制图即使"看起来正常"也**不能放过**。

#### 相关操作

- **Cpk 月报**：报表 → SPC 月报，含**每张图的 Cpk 趋势 + 排名**。
- **AI 工艺建议**：AI 助手问「某控制图 Cpk 为什么这个月下降」可获得**因果分析**。

## 7. 拍照自动计数

> **价值**：拍一张产品照片，**AI 自动数出零件数**，免去手动数。

- **使用者**：操作员工
- **操作路径**：H5 → 报工页 → 「**AI 数一下零件**」按钮

#### 操作步骤

1. 进入报工页（扫码或选任务）。
2. 在「**合格数**」输入框右侧点「**AI 数一下零件**」按钮。
3. 弹出相机 → **拍产品正面**（光线好、零件不重叠）。
4. 等待 1~3 秒，AI 识别完成后：
   - **自动填入合格数**
   - 显示**置信度**：高（绿色）/ 中（黄色）/ 低（红色，需人工复核）
5. 复核无误 → 继续报工。

#### 适用与不适用

| ✅ 适合 | ❌ 不适合 |
| --- | --- |
| 标准件散落在桌面 | 重叠 / 堆叠 |
| 颜色 / 形状一致 | 多种零件混放 |
| 光线均匀 | 强反光 / 强阴影 |

#### 注意事项

- AI 数的是**「照片中能识别的数量」**，不是「你今天的合格数」。**两个概念不要混**。
- **置信度低**时建议人工数 + 在备注里说明。
- 支持**一次提交多张照片**（系统取**识别最多**的那张）。

#### 相关操作

- **历史记录**：「我的 → AI 识别记录」可看所有 AI 数过的图。
- **纠错反馈**：识别错误时点「**纠错**」→ 帮 AI 学习（需启用反馈学习）。

## 8. 语音报工

> **价值**：边干活边说话，**AI 自动解析"做了 50 个好的，2 个有划痕"** → 填入表单。

- **使用者**：操作员工
- **操作路径**：H5 → 报工页 → 备注框旁「**🎙 录音**」按钮

#### 操作步骤

1. 进入报工页。
2. 准备录入时，点备注框旁的「**🎙 录音**」按钮（首次会弹权限请求：允许使用麦克风）。
3. 说出内容（参考话术见下表）。
4. 说完后再次点「**🎙 停止**」。
5. AI 解析后：
   - **good_qty（合格数）** 自动填入
   - **bad_qty（不良数）** 自动填入
   - **defect_keywords（缺陷关键词）** 自动填入备注
6. 核对 → 继续提交。

#### 推荐话术

| 场景 | 话术 |
| --- | --- |
| 只报数量 | "50 个好的" |
| 报合格+不良 | "做了 50 个好的，2 个有划痕" |
| 报不良品 | "3 个是次品，变形" |
| 报进度 | "今天到第 80 件" |

#### 注意事项

- **需要 HTTPS 环境**（麦克风权限限制）。
- 单次录音**最长 60 秒**，超长内容分段录。
- 方言 / 噪声严重时识别率下降，建议**标准普通话**。
- 解析后的数字**仍要人工核对**（特别是"一"和"七"等音近字）。

#### 相关操作

- **语音 + 拍照** 可同时使用：先 AI 数 → 再语音补充不良信息。
- **手动覆盖**：AI 解析后任何字段都可**手动改**，AI 不会重复触发。

## 9. AI 自动审核

> **价值**：低风险报工**自动通过**，省去班组长 50% 的审核工作。  
> AI 从 6 个维度评估：**良率、员工熟练度、照片质量、缺陷严重度、工序、时段**。

- **使用者**：班组长（被替代部分工作）/ 系统（自动处理）
- **操作路径**：
  - **PC**：Admin-Pro → 生产 → 报工审核 → 看「**AI 预审**」标签
  - **H5**：报工提交后立即看到 AI 提示

### 9.1 自动通过（低风险）

#### 触发条件

- 风险评分 **< 20**（满分 100，越低越安全）
- 典型场景：熟练员工的常规报工 + 良率正常 + 照片清晰

#### 体验

- 员工 H5 提交后，**秒级收到推送**：「**AI 已自动审核通过**」。
- 报工**直接生效**到工资计算，**无需人工审核**。
- 管理员可在「**报工审核 → AI 通过**」tab 查看（仅供审计）。

#### 注意事项

- AI 通过**不代表无错**，管理员可**事后抽查**（建议每周抽 5%）。
- 「**报工审核 → AI 审核规则**」可调整风险阈值（默认 20）。

---

### 9.2 正常 / 强制人工（中 / 高风险）

| 风险等级 | 评分 | 体验 |
| --- | --- | --- |
| **中风险** | 20 ~ 59 | 报工**提交给班组长正常审核**，AI 给出**风险点提示**（如"良率比上周低 5%"） |
| **高风险** | ≥ 60 | **强制人工审核** + 弹窗提示（员工也会看到："您的报工已转人工"），并**推送给车间主任** |

#### 班组长审核页 AI 辅助

- 报工列表每条带「**AI 预审**」徽章：
  - 🟢 **低风险**：可一键通过
  - 🟡 **中风险**：查看 AI 给出的原因
  - 🔴 **高风险**：必须重点看照片 / 视频，**不可一键通过**
- 鼠标悬停徽章，**展开 6 维度雷达图**（一键看懂哪里可疑）。

#### 6 维度评分（参考）

| 维度 | 说明 |
| --- | --- |
| 良率 | 低于该员工历史均值则加分 |
| 熟练度 | 新员工 / 转岗员工加分 |
| 照片质量 | 模糊 / 缺照片 / 角度异常加分 |
| 缺陷严重度 | 不良数 > 阈值则加分 |
| 工序 | 高风险工序（如焊接）加分 |
| 时段 | 深夜 / 加班时段加分 |

#### 注意事项

- AI 自动通过记录**保留完整证据**（照片 / 视频 / 各项评分），可随时审计。
- 管理员可在「**系统 → AI 设置**」单独关闭某个维度的评分。
- 触发**高风险**会**自动抄送车间主任**（即使你只是班组长）。

## 10. AI 缺陷分类

> **价值**：拍一张不良品照片，AI 自动识别**是什么缺陷**（划痕 / 变形 / 气泡...），自动填入备注。

- **使用者**：操作员工 / 质检员
- **操作路径**：H5 → 报工页（填不良数时）→ 「**📷 识别缺陷**」按钮

#### 操作步骤

1. 报工时填了「不良数 > 0」后，出现「**📷 识别缺陷**」按钮。
2. 点击 → 弹出相机 → **拍不良品特写**（对准缺陷部位、对焦清晰）。
3. 等待 1~3 秒，AI 识别：
   - **缺陷代码**（如 `SCRATCH` 划痕）
   - **缺陷名称**（中文）
   - **置信度**（高 / 中 / 低）
4. 自动填入备注框 → 你可**手动补充**细节（如"在左下角"）。
5. 提交报工。

#### 缺陷库

- 系统预置 **30+ 常见缺陷**（划痕、变形、气泡、杂质、缺口、错位...）。
- 管理员可在「**主数据 → 缺陷代码**」**自定义缺陷码**。
- 报工后 AI 也会把新缺陷**加入学习**（需开启反馈学习）。

#### 注意事项

- AI 识别的缺陷会**自动同步到「缺陷分析报表」**，可做 TOP 10 排名。
- 置信度低时建议**手动选缺陷码**（从下拉列表）。
- **多缺陷**照片可重复识别（每次识别后追加到备注）。

#### 相关操作

- **缺陷趋势**：报表 → 缺陷分析 → 看某缺陷**最近 30 天趋势**。
- **缺陷 - 员工关联**：报表 → 缺陷分析 → 「按员工分组」看哪类员工高频出某缺陷。

## 11. 智能报工建议

> **价值**：打开 H5，**AI 主动告诉你"今天该报哪个"**——不用自己翻任务列表。

- **使用者**：操作员工
- **操作路径**：H5 → 主页顶部「**💡 智能推荐**」紫色卡片

#### 操作步骤

1. 登录 H5 → 主页。
2. 顶部出现紫色「**💡 智能推荐**」卡片，显示 Top 3 推荐任务。
3. 每条推荐带**推荐理由**（可点 "?" 查看）：
   - 「**距上次报工 2 天，剩 50 件**」（最紧急）
   - 「**你擅长的工序**」（熟练度高）
   - 「**新到任务，金额 ¥120**」（高单价）
4. 点某条任务 → 直接跳到报工页。
5. 不感兴趣可点「**换一批**」→ 重新推荐。

#### 推荐算法综合因素

| 因素 | 权重 | 解释 |
| --- | --- | --- |
| 紧急度 | 高 | 交期 / 剩余数量 |
| 熟练度 | 中 | 历史报工数据 |
| 金额 | 中 | 工价 × 预计件数 |
| 时段 | 低 | 适合当前时间的工序 |

#### 注意事项

- 推荐**不替代正常任务列表**；不想用推荐可点「**查看全部任务**」。
- 报工完成后，**推荐自动刷新**（下次进入主页时）。
- 管理员可关闭推荐：「**系统 → AI 设置**」。

#### 相关操作

- **历史推荐**：「我的 → 推荐历史」可看最近 30 天的推荐 + 你点了哪些。
- **调整权重**：AI 助手对话可反馈"我更想看高单价的"，AI 会**记住你的偏好**。

## 12. 换班交接摘要

> **价值**：交接班时不用口述半天，**AI 自动生成摘要**：「本班做了 45 单 120 件，发现 3 次模具预警，1 个工单延期风险」。

- **使用者**：班组长 / 车间主任
- **操作路径**：H5 → 主页 → 「**📋 交接摘要**」卡片 → 「**生成摘要**」

#### 操作步骤

1. 主页找到「**📋 交接摘要**」卡片（首页中部偏下）。
2. 确认**班次起止时间**（默认 8 小时前到现在）。
3. 点「**生成摘要**」按钮。
4. AI 在 5~10 秒内生成结构化摘要：
   - **产量**：本班次报工单数 / 件数 / 工时
   - **质量**：合格数 / 不良数 / 良率
   - **订单**：完成 / 在制 / 延期预警
   - **异常**：模具预警 / 设备故障 / 物料不足
   - **建议**：下个班次重点关注什么
5. 摘要可：
   - **复制**到剪贴板（发群 / 邮件）
   - **推送**到飞书 / 钉钉群
   - **保存**为 PDF 存档

#### 摘要内容示例

> **本班次（2026-06-15 08:00 ~ 16:00）摘要**  
> - 报工：**45 单 / 120 件 / 8.5 工时**（达成率 95%）  
> - 良率：**96.5%**（较上日 ↑1.2%）  
> - 完成订单：**3 单**（D-20260612001, D-20260613005, D-20260614002）  
> - 延期预警：**1 单**（D-20260611008 交期 6/16，需下个班次加急）  
> - 模具预警：**2 次**（MOLD-005 寿命 92%, MOLD-007 寿命 85%）  
> - 异常报警：无  
> - **建议**：下个班次优先冲 MOLD-005 的 K005 工单，避免交期逾期。

#### 注意事项

- 摘要生成**需要联网**（调用 AI 接口）。
- 摘要**不是流水账**，是**重点 + 建议**，适合直接发班组长群。
- 同一班次可**多次生成**，以最新一次为准。

#### 相关操作

- **定时自动生成**：可在「**系统 → AI 设置**」开启「**交接班前 30 分钟自动生成**」。
- **工厂日报**：每天 20:00 自动生成日报并推送管理群（详见 in-app help「**12.5 AI 助手**」）。

## 附录 A 常见问题

### Q1：报表导出任务一直"处理中"怎么办？

1. 刷新「**导出中心**」页面看最新状态。
2. 等 30 分钟仍"处理中"：
   - 联系系统管理员查**后端 Celery worker 是否在跑**。
   - 提供任务 ID 让管理员查日志：`/tmp/lightmes-celery/worker.log`。
3. 状态为「**失败**」时：先「**重试**」一次；仍失败**重建任务**（缩小时间范围 / 用 CSV 格式）。

### Q2：Lodop 打印没反应？

1. 确认浏览器是 **Chrome / Edge / 360**（不支持 Firefox / Safari）。
2. 检查浏览器右上角是否**拦截了 Lodop 插件** → 点"允许"。
3. 重新安装 Lodop 控件：系统管理 → 打印设置 → 下载 → 安装 → **重启浏览器**。
4. 仍失败改用 **ZPL 网络打印**（不受 Lodop 限制）。

### Q3：模具寿命预警频繁推送？

- 寿命阈值设置过低（默认 80%）：在「**系统 → 模具参数**」调到 90%。
- 某些模具**快到达寿命但不影响生产**（如设计保守）：在预警记录里「**标记误报**」，后续不再推。

### Q4：SPC 控制图全红，但 Cpk 还 1.5？

- 控制图异常 ≠ 过程能力不足。可能最近几批**设备有调整**（如换了刀）。
- 处理：检查 4M 变化（人 / 机 / 料 / 法），做过程验证后再判断。
- 控制图只反映"现在"，Cpk 反映"过去"——**两者结合看**最准。

### Q5：审批流改完之后历史报工怎么办？

- **不影响**！在途审批仍按**老流程**继续。
- 切换时间点：新提交的报工**才走新流程**。
- 想"统一用新流程"？需要**等所有在途审批走完**再改。

### Q6：PWA 离线队列一直提交失败？

- 90% 是**任务已被别人报完**：进入报工详情看状态。
- 其他可能：网络问题（切换 WiFi / 数据流量）；任务**已被作废**。
- 「**重试**」3 次仍失败 → 「**删除**」该条并联系班组长手动补录。

### Q7：AI 数零件数和实际对不上？

- 检查照片：是否**重叠 / 反光 / 角度异常**（见 [7. 拍照自动计数](#7-拍照自动计数) 的"适用与不适用"表）。
- 拍照时**铺平 / 散开 / 自然光**最佳。
- AI 只能数**照片里能看到的**，不是真"做出来的"——以你**实际合格数**为准。

### Q8：语音报工解析错了（"一"被识别成"七"）？

- 改用**标准数字读法**："一 → 幺" / "二 → 两" / "七 → 拐"（工厂常用法）。
- 解析后**人工核对**（必做）。
- 反复错误的字段可**手填**覆盖，AI 不会重复触发。

### Q9：AI 自动审核把我的报工"误判"高风险？

- 高风险 ≠ 错误，**是更谨慎**。审核员会重点看，没问题正常通过。
- 如果**总是**高风险：检查照片是否清晰、是否按时报工、有无不良率波动。
- 可在「**我的 → AI 审核记录**」看每次的 6 维度雷达图，**有针对性改进**。

### Q10：换班摘要生成失败 / 内容不准？

- 联网问题：换班摘要需联网调 AI。
- 内容不准：摘要基于**最近 N 小时报工数据**，确认你的报工**已提交**（在「**我的报工**」可查）。
- 摘要仅供参考，**重要决策请结合系统数据 + 现场观察**。
