# LightMes 套餐体系规划方案

> 基于现有 SaaS 平台（套餐管理、租户管理、订阅订单、讯虎支付）及行业包、社区版/Pro 版拆分的全面重新规划

---

## 一、现状分析

### 当前数据库
| 项目 | 内容 |
|------|------|
| 套餐数量 | **1 个**（测试租户 - 0元/7天/10用户） |
| 实际定价 | 无正式定价 |
| 行业包 | 已开发 6 大行业包，但**未与套餐绑定** |

### 现有体系问题
1. **套餐层级缺失** — 只有1个测试套餐，没有正式定价结构
2. **社区版限制未实施** — pro-manifest.txt 和 OpenCore 文档列了，但代码里 `LIGHTMES_EDITION` 环境变量、硬限制中间件都还没实现（文档自己也标注了"尚未实现"）
3. **行业包未融入套餐** — 行业包是独立功能，没有与套餐层级挂钩
4. **注册流程不完整** — 注册页可以选择套餐，但套餐数据和展示页面还未完善
5. **平台管理页功能弱** — 套餐管理目前只支持新增/编辑，缺乏完整的发布/下架管理

---

## 二、定价层级设计（3层 + 开源）

| 层级 | 定位 | 定价参考 | 核心用户 |
|------|------|---------|---------|
| **Community（开源）** | 扫码报工工具，引流入门 | **免费** | 小微企业、个人开发者 |
| **Starter（入门版）** | 轻量SaaS，自助开通 | **¥ 998/年** | 小型加工厂（5-15人） |
| **Pro（专业版）** | 完整经营系统 | **¥ 3,980/年** | 中型加工厂（15-100人） |
| **Enterprise（企业版）** | 全功能+私有部署 | **¥ 15,000/年起** | 大型工厂、集团 |

## 四、行业包定价策略

行业包是**差异化核心卖点**，建议：

| 策略 | 说明 |
|------|------|
| **Starter** | 激活时**免费选1个**行业包（激活后不可换） |
| **Pro** | **最多激活3个**行业包，随时切换 |
| **Enterprise** | **6个行业包全开**，无限制 |
| **单购行业包** | 可额外购买¥ 500/个/年（针对需要多行业的Starter用户） |
| **上架更多行业包** | 未来可扩展付费行业包（如¥ 2,000/个/年），作为增值收入 |

## 五、建议实施步骤

### 阶段一：整理现有套餐数据
1. 在平台后台创建正式套餐（Community免创建，自动识别）
2. 给演示租户设置有效套餐
3. 清理掉测试套餐 `PKG202606150001`

### 阶段二：代码改造
1. **添加 `features_json` 标准格式** — 每个套餐的 features_json 存储该套餐包含的功能清单（权限点列表）
2. **套餐与行业包关联** — Tenant 的 `industry_codes` 字段受 `max_industry_packs` 限制
3. **套餐对比页** — 官网 `/pricing` 显示3个层级的定价和功能对比

### 阶段三：平台管理增强
1. 套餐表单增加行业包数量字段
2. 套餐自动续费机制
3. 套餐升降级流程

### 阶段四：前端展示
1. 官网增加定价页面
2. 注册流程增加套餐选择步骤
3. 管理后台增加"当前套餐"概览页

## 六、代码改造要点

### 1. SaasPackage 模型新增字段
```python
max_industry_packs: int = 1  # 允许激活的行业包数量
max_storage_gb: int = 1      # 存储空间上限(GB)
features_json  # 已有，标准化格式存功能权限点列表
```

### 2. 套餐校验中间件
```python
# 每个API请求前校验当前租户的套餐限制
# - max_users → 查询用户表
# - max_industry_packs → 激活行业包数量
# - 报工条数限制 → 月度统计
```

### 3. 注册流程修改
```python
# /api/v1/tenants/register 增加套餐选择
# 选择付费套餐 → 创建订单 → 引导支付 → 激活
# 选择免费试用 → 默认分配 Starter 套餐14天试用
```
