# 报工与溯源形态（多形态说明）

LightMes 支持按租户配置默认报工形态，适应不同车间；**后期可按企业单独扩展**（如批次码规则、定制溯源字段）。

## 形态一览

| 形态 | 配置值 | 状态 | 适用场景 |
|------|--------|------|----------|
| **批量报工** | `batch` | ✅ 默认 | 班长派工、员工扫任务码填合格/不良数；大批量、分工数量不同 |
| **逐件报工** | `unit` | ✅ 可选 | 件次槽位、现场拍照/视频、首工序成品码 FP、工序流转链 |
| **批次流转** | `lot` | 🔜 预留 | 一扎/一筐一批次码跟货（后期定制） |

## 配置位置

管理后台 → **系统设置** → **报工形态（全厂默认）**

对应租户键：`production.report.default_mode`

未配置时默认为 `batch`。

## 员工端行为

- **batch**：首页 / 扫码报工 → 输入数量；按 `task_assignments` 派工上限校验；不计件溯源成品码。
- **unit**：扫任务码后进入 **逐件报工**；**首道工序派工即预建套号池并赋 FP 成品码**；员工 **谁有空谁领下一套**（按 `piece_no` 升序自动分配，不再固定「员工 A 只做 1–15 套」）；后道自动绑定上道已终审通过的套号；客户扫 **每一套的 FP** 溯源。
- **lot**：暂未开放，保存时会提示。

## 溯源查询

- **batch**：按订单号、报工记录、任务码查审核与产量（管理端报工/工资/订单进度）。
- **unit**：溯源页扫 `FP...` 或历史 `TC...` 查看工序流转链。
- 管理端菜单仍保留「件次报工审核」「历史批量报工」与形态并行，便于过渡。

## 逐件模式（unit）套号池

1. **工单 45 套** → 预建 `WorkOrderPiece` 1..45，派工首道工序时赋 `FP...`（`status=reserved`）。
2. **木工 3 人、各派 15 件** → 三人共享同一套号池，提交报工时从池中 **领取下一可用套**。
3. **后道工序** → 只能领 **上道已终审合格** 且本道尚未报的套。
4. **首道不良终审** → 该套标记 `void`，不再进入后续工序。
5. **发货绑柜**（45 套装柜等）→ 二期在发货单明细绑 FP，与生产套号一致。

API 字段：`flow.piece_pool_enabled`、`pool_total`、`pool_available`；报工返回 `piece_no`、`product_code`、`unit_label`。

三种形态均共用现有 **任务 + 派工（不同员工、不同数量）** 模型；差异仅在员工 **如何报工** 与 **是否生成件次/成品码**。

## 后期扩展建议

- 按 **产品 / 工序** 覆盖默认形态  
- 批次模式：`LOT` 码 + 流转卡打印  
- 混线：同一租户多车间不同形态（需组织架构级配置）
