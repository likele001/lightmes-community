# LightMes 部署说明（通用 Linux / Windows，无 Docker）

## 先选部署模式

| | 路线 A：单厂私有化 | 路线 B：SaaS 商业化 |
|---|-------------------|---------------------|
| **场景** | 一家厂自用 | 对外卖账号、多工厂 |
| **build** | admin-pro + h5 | admin-pro + h5 + **portal** |
| **`/platform`** | 可不用 | **必须用**（运营商） |
| **`/register`** | 不部署 | 部署 portal |
| **dev 还是 build** | 生产环境只用 **`npm run build`** | 同上 |

- **路线 A**：[单厂私有化](#路线-a单厂私有化) — 做完「后端 + 前端部署」即可  
- **路线 B**：[SaaS 商业化](#路线-bsaas-商业化) — 在 A 的基础上增加平台、portal、支付  

### `npm run build` 要跑几个目录？（速查）

| 目录 | 路线 A | 路线 B | build 后挂到 |
|------|:------:|:------:|--------------|
| `frontend-admin-pro` | 要 | 要 | 管理端站点根目录（含 `/platform`） |
| `frontend-h5` | 要 | 要 | H5 站点根目录 |
| `frontend-portal` | 不要 | 要 | 注册站点根目录（SaaS 自助注册） |
| `backend` | **不用 npm** | **不用 npm** | 宝塔 Python 项目，端口 8000 |

```bash
# 路线 A：2 个前端
cd frontend-admin-pro && npm install && npm run build
cd frontend-h5 && npm install && npm run build

# 路线 B：再加 portal
cd frontend-portal && npm install && npm run build
```

更细的宝塔站点、伪静态、反代、URL 对照见 **[宝塔部署.md → 速查](宝塔部署.md#速查npm-run-build-要跑几个目录最易乱先看这里)**。

## 运行环境要求

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| Python | 3.10+ | 后端运行环境 |
| Node.js | 18+ | 前端构建/开发 |
| MySQL | 5.7+ | 数据库 |
| Redis | 6.x+ | Celery 异步任务（可选，不影响核心业务） |
| Nginx | 任意 | 生产环境反向代理（可选） |

## 环境准备

### Debian / Ubuntu

```bash
sudo apt update
sudo apt install -y python3 python3-venv python3-pip build-essential
sudo apt install -y mysql-server mysql-client
sudo apt install -y redis-server
python3 --version
```

Node.js LTS：

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
node -v
npm -v
```

### Windows

- Python 3.10+：从 [python.org](https://python.org) 下载安装
- Node.js 18+：从 [nodejs.org](https://nodejs.org) 下载安装
- MySQL 5.7+：建议使用 [XAMPP](https://www.apachefriends.org/) 或 [MySQL Installer](https://dev.mysql.com/downloads/installer/)
- Redis：Windows 下建议使用 [Memurai](https://www.memurai.com/)（兼容 Redis）或 WSL

## 后端部署

### 1. 配置环境变量

```bash
cd backend
cp .env.example .env
```

编辑 `.env`，必须修改：

```ini
# MySQL 连接串（替换用户名/密码/主机）
DB_URL=mysql+pymysql://用户名:密码@127.0.0.1:3306/lightmes?charset=utf8mb4

# JWT 密钥（必须修改为随机字符串，不要用默认值）
JWT_SECRET=your-random-secret-here

# Redis 连接（如果不用 Celery 可保留默认值）
REDIS_URL=redis://127.0.0.1:6379/0
```

### 2. 安装依赖

```bash
cd backend
python3 -m venv .venv            # 创建虚拟环境（可选但推荐）
source .venv/bin/activate         # Linux
# 或 .venv\Scripts\activate       # Windows
pip install -r requirements.txt
```

### 3. 数据库迁移

> **宝塔 / 生产环境**：若提示 `alembic: command not found`，请使用 Python 虚拟环境全路径，详见 [`docs/宝塔部署.md`](宝塔部署.md) **二、2.2**。

**本地开发（项目 `.venv`）：**

```bash
cd backend
source .venv/bin/activate    # Windows: .venv\Scripts\activate
pip install -r requirements.txt
alembic upgrade head
alembic current              # 可选：查看当前版本
```

**宝塔生产（示例路径，以你面板里「环境路径」为准）：**

```bash
cd /www/wwwroot/lightmes/backend
/www/server/pyporject_evn/hightmes/bin/pip install -r requirements.txt
/www/server/pyporject_evn/hightmes/bin/alembic upgrade head
```

迁移成功后，数据库中会自动创建/更新所有表（含 `code_sequences` 等业务编号序号表）。完成后**重启**后端服务。

登录有效期（`backend/.env`）：未勾选「记住登录」为 **8 小时**（`ACCESS_TOKEN_EXPIRE_MINUTES=480`）；勾选为 **7 天**（`REMEMBER_ME_EXPIRE_MINUTES=10080`）。

### 4. 启动后端

```bash
cd backend
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

访问 `http://127.0.0.1:8000/api/health` 验证。

## 前端部署

### 开发模式

```bash
# PC 管理端
cd frontend-admin-pro
npm install
npm run dev                       # http://localhost:5173

# H5 移动端
cd frontend-h5
npm install
npm run dev                       # http://localhost:5174
```

### 生产构建（Nginx 托管）

```bash
# 构建 PC 管理端
cd frontend-admin-pro
npm install
npm run build                     # 产物在 dist/ 目录

# 构建 H5 移动端
cd frontend-h5
npm install
npm run build                     # 产物在 dist/ 目录
```

Nginx 配置要点：

```nginx
# 管理端
server {
    listen 80;
    server_name admin.yourdomain.com;
    root /path/to/frontend-admin-pro/dist;
    index index.html;
    location / { try_files $uri $uri/ /index.html; }
}

# H5
server {
    listen 80;
    server_name h5.yourdomain.com;
    root /path/to/frontend-h5/dist;
    index index.html;
    location / { try_files $uri $uri/ /index.html; }
}

# API 反向代理
server {
    listen 80;
    server_name api.yourdomain.com;
    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

前端通过 `/api/` 路径请求后端，或在前端站点 Nginx 中反向代理 `/api` 到后端地址。

## 路线 A：单厂私有化

### 部署什么

- `backend` + `frontend-admin-pro`（build）+ `frontend-h5`（build）
- **不** 部署 `frontend-portal`

### 生产环境 URL

| 角色 | 地址 |
|------|------|
| 工厂管理员 | `https://admin域名/login`（租户编码 `DEMO`）或 `/t/DEMO/admin` |
| 员工 / 客户 | `https://h5域名/t/DEMO/h5` |

### 开发调试（可选）

```bash
bash start.sh all    # 后端 + admin dev + h5 dev
# 不要用 dev 上宝塔，宝塔用 npm run build
```

## 路线 B：SaaS 商业化

### 部署什么（在路线 A 的 build 基础上增加）

| 组件 | 说明 |
|------|------|
| `frontend-portal` | **必须** build，单独 Nginx 站点 → `/register/` |
| 平台账号 | `PYTHONPATH=. python scripts/init_platform_admin.py platform 密码` |

### 生产环境 URL（运营商 vs 工厂）

| 角色 | 地址 |
|------|------|
| **运营商（你）** | `/platform/login` → `/platform/tenants`（租户管理） |
| 客户注册 | `https://www域名/register/` |
| 某厂管理员 | `/t/厂编码/admin` |
| 该厂员工 | `/t/厂编码/h5` |

### portal 构建

```bash
cd frontend-portal
npm install
npm run build    # 生产
# npm run dev    # 仅本机调试 :5175
```

### 平台设置

- **关闭售卖租户**：仅平台手工建厂，不开放 `/register`
- **开启售卖租户**：开放注册 + 虎皮椒（见宝塔文档 B.6 节）

### 租户路径、支付、迁移问题

详见 [`宝塔部署.md`](宝塔部署.md) 中 **路线 B** 章节（B.5～B.9）。

如果启用了 Redis，可启动 Celery worker 处理异步任务（工资导出、CRM 公海池回收等）：


## 常见问题

### 后端无法连接 MySQL
- 检查 MySQL 是否已启动：`systemctl status mysql`
- 检查 `.env` 中的 `DB_URL` 用户名/密码/主机是否正确
- 检查 MySQL 用户是否有远程/本地访问权限
- 确保数据库 `lightmes` 已创建

### 前端刷新 404
前端为 SPA 应用，需要在 Nginx 中配置：
```nginx
location / { try_files $uri $uri/ /index.html; }
```

### 前端调接口 401
- 确认用户已登录（获取了 token）
- 确认 HTTP 请求头中携带了 `Authorization: Bearer <token>`
- 检查 `.env` 中的 `JWT_SECRET` 是否与登录时一致

### 迁移执行失败

**`alembic: command not found`**：使用虚拟环境全路径，例如：

```bash
ALEMBIC=/www/server/pyporject_evn/hightmes/bin/alembic   # 改成你的环境
cd /www/wwwroot/lightmes/backend
$ALEMBIC history
$ALEMBIC current
$ALEMBIC upgrade head
```

**表已存在、版本错乱**：见 [`docs/宝塔部署.md`](宝塔部署.md) **B.8**，使用 `alembic stamp 0027_code_sequences`（或仓库最新 revision）。
