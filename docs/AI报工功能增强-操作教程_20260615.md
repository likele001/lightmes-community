## 2. 功能 1：拍照自动计数

### 2.1 后端接口

**接口**：`POST /api/h5/ai/report/photo-count`

**权限**：`ai.report_assist`

**请求体**：
```json
{
  "image_urls": ["/api/files/1?download=true", "/api/files/2?download=true"],
  "task_id": 123,
  "hint": "这是圆形零件，数有多少个"
}
```

**返回**：
```json
{
  "ok": true,
  "count": 12,
  "confidence": "high",
  "per_image": [6, 6],
  "note": "每张图目视 6 个，共 12 个",
  "image_count": 2
}
```

### 2.2 cURL 测试

```bash
VENV=/www/server/pyporject_evn/lightmes
TOKEN="<员工的登录 token>"

curl -X POST "http://127.0.0.1:8000/api/h5/ai/report/photo-count" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"image_urls": ["/api/files/1?download=true"], "task_id": 123}'
```

### 2.3 前端操作（ReportWorkPage）

1. 进入「扫码报工」页面
2. 扫描或输入任务码
3. 上传照片（点击"添加文件"按钮拍摄/选择）
4. 点击 **"AI 数一下零件"** 按钮
5. AI 自动识别数量并填入「合格数」输入框
6. 用户确认后提交报工

## 3. 功能 2：语音报工

### 3.1 后端接口

**接口**：`POST /api/h5/ai/report/voice-parse`

**权限**：`ai.report_assist`

**请求体**：
```json
{
  "text": "我做了50个好的，2个不良，其中一个有划痕",
  "task_id": 123
}
```

**返回**：
```json
{
  "good_qty": 50,
  "bad_qty": 2,
  "result_type": "mixed",
  "remark": "缺陷关键词: 划痕",
  "defect_keywords": ["划痕"]
}
```

### 3.2 cURL 测试

```bash
curl -X POST "http://127.0.0.1:8000/api/h5/ai/report/voice-parse" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"text": "做了60个全部合格", "task_id": 123}'
```

### 3.3 前端操作（ReportWorkPage）

1. 进入「扫码报工」页面
2. 扫描或输入任务码
3. 点击备注右侧的 **"按住说话"** 按钮（蓝色，录音时变红色）
4. 说一段话描述报工内容，如："我今天做了60个，全部合格"
5. 再次点击停止录音（或自动停止）
6. 前端自动调用 voice-parse API 解析
7. 解析结果自动填入合格数/不良数/备注
8. 用户修改确认后提交

> **注意**：Web Speech API 需要 HTTPS 环境（localhost 除外）  
> iOS Safari 14.5+ / Android Chrome 均支持

## 4. 功能 3：AI 自动审核

### 4.1 后端接口

无需单独 API。在报工提交 API `POST /api/h5/report-units` 中**自动触发**。

### 4.2 评分逻辑

| 维度 | 权重 | 计算方式 |
|------|------|----------|
| 历史良率 | 25分 | 近 30 天良率 <80% 扣25分；<95% 扣10分 |
| 历史驳回率 | 20分 | 驳回率 >10% 扣20分 |
| 入职时长 | 10分 | 入职 <30 天扣10分 |
| 报工数量 | 15分 | 超过剩余×1.5 扣15分 |
| 照片证据 | 10分 | 无照片扣10分 |
| 不良率突增 | 20分 | 近期不良突增 >20% 扣20分 |

### 4.3 提交后响应示例

```json
{
  "ok": true,
  "data": {
    "id": 456,
    "status": "leader_approved",
    "risk_score": 15.0,
    "risk_level": "low",
    "auto_passed": true,
    "risk_reasons": ["缺乏足够历史数据"]
  }
}
```

### 4.4 前端效果

- **低风险** → 提交后 Toast 显示 **"AI 已自动审核通过"**
- **中风险** → 显示 **"已提交班组长审核"**
- **高风险** → 显示 **"⚠️ 高风险报工，已提交班组长审核"**

### 4.5 风险评分单独测试

```python
VENV=/www/server/pyporject_evn/lightmes
$VENV/bin/python -c "
from app.services.ai.report_risk_score import calculate_risk_score, _level_for_score
# 直接测试阈值
for s in [0, 10, 19, 20, 50, 59, 60, 80, 100]:
    print(f'score={s:3} → level={_level_for_score(s)}')
"
```

## 5. 功能 4：AI 缺陷自动分类

### 5.1 后端接口

**接口**：`POST /api/h5/ai/report/defect-classify`

**权限**：`ai.report_assist`

**请求体**：
```json
{
  "image_urls": ["/api/files/1?download=true"],
  "task_id": 123,
  "remark": "外观有划痕"
}
```

**返回**：
```json
{
  "ok": true,
  "defect_code_id": 5,
  "defect_code": "SCRATCH",
  "defect_name": "划痕",
  "severity": "major",
  "confidence": "high",
  "description": "表面可见划痕，长度约3cm",
  "image_count": 1,
  "options_count": 12
}
```

### 5.2 前端操作（逐件模式-ReportUnitPage）

1. 进入件次报工页面
2. 选择 **"不良"**
3. 上传照片
4. 点击 **"AI 识别缺陷类型"** 按钮
5. AI 识别后自动填入备注（如 `【划痕】严重度:major / 表面可见划痕`）
6. 用户修改确认后提交

### 5.3 前端操作（批量模式-ReportWorkPage）

1. 进入批量报工页面
2. 填写不良数 > 0
3. 上传照片
4. 点击 **"AI 识别缺陷"** 按钮
5. AI 识别结果自动填入备注

## 6. 功能 5：智能报工建议

### 6.1 后端接口

**接口**：`GET /api/h5/ai/report/recommend?top=3`

**权限**：`ai.report_assist`

**返回**：
```json
{
  "items": [
    {
      "task_id": 1,
      "task_code": "TK-20260601-001",
      "process_name": "CNC 加工",
      "remaining_qty": 50,
      "assigned_qty": 100,
      "reported_qty": 50,
      "priority": "urgent",
      "last_report_at": "2026-06-13T10:00:00",
      "reason": "距上次报工 2 天 · 剩 50 件 · 1 天后交期"
    }
  ]
}
```

### 6.2 cURL 测试

```bash
curl -X GET "http://127.0.0.1:8000/api/h5/ai/report/recommend" \
  -H "Authorization: Bearer $TOKEN"
```

### 6.3 前端操作（TasksPage）

1. 进入「我的任务」页面
2. 页面顶部显示 **紫色渐变 AI 推荐卡片**
3. 卡片展示 Top 3 推荐任务 + 推荐理由
4. 点击任一推荐项直接跳转到对应报工页面
5. 推荐标签：**「急」**（交期≤3天）或 **「推荐」**

> 推荐算法基于：上次报工时间 / 剩余数量 / 订单交期 / 工序顺序 / 任务状态，**无需 AI 模型，毫秒级响应**。

## 8. Admin-审核列表风险标签

### 8.1 API 变更

Admin 端 `GET /api/admin/report-units` 新增字段：

```json
{
  "items": [
    {
      "id": 123,
      "status": "submitted",
      "prescreen_level": "high",
      "prescreen": {
        "level": "high",
        "score": 75.0,
        "reasons": ["近 30 天良率 72%，明显偏低", "入职仅 15 天，新员工"],
        "auto_pass": false
      }
    }
  ]
}
```

### 8.2 新查询参数

| 参数 | 类型 | 说明 |
|------|------|------|
| `prescreen_level` | string | 按风险等级过滤：`low` / `medium` / `high` |
| `risk_first` | bool | 高风险优先排序（默认 false） |

### 8.3 前端建议（Admin-Pro）

Admin 审核列表可基于 `prescreen.level` 做：
- `high` → **红色**标签 🔴
- `medium` → **黄色**标签 🟡  
- `low` → **绿色**标签 🟢
- `null` → 灰色标签（未评估）

## 9. 常见问题排查

### Q1: AI 报工提示"AI 暂不可用"

**排查步骤**：
```bash
# 1. 检查 AI 平台配置
curl -X GET "http://127.0.0.1:8000/api/h5/ai/models" \
  -H "Authorization: Bearer $TOKEN"

# 2. 检查租户 AI 配置（确保 api_key 已设置）
# 3. 检查 vision_completion() 的提示词是否正确
```

### Q2: 语音不工作

```bash
# 1. 检查是否 HTTPS（Web Speech API 要求安全上下文）
# 2. 检查麦克风权限（浏览器设置）
# 3. 检查 isVoiceInputSupported()
npx tsx -e "
import { isVoiceInputSupported } from './src/utils/voice-input'
console.log('Voice input supported:', isVoiceInputSupported())
"
```

### Q3: 风险评分未生效

```bash
# 1. 检查提交响应是否返回 risk_score 字段
# 2. 检查 unit.prescreen_level 字段是否已写入
# 3. 确认 user 是否有 'ai.report_assist' 权限
# 4. 检查日志：app.crud.automation_log 表中 action='report_risk_score_error'
```

### Q4: 推荐排序不准确

检查 `app/services/ai/task_recommend.py` 评分逻辑，确认：
- 员工有派工记录（TaskAssignment）
- 派工任务有关联 WorkOrder → Order（获取 due_date）
- 报工记录通过 ReportUnit.unit_seq 累加
