# LightMes 三站点前端宝塔配置方案

> 管理端 / 客户端（H5）/ 注册端 独立部署说明。  
> 适用域名示例：`admin.lightmes.user.023ent.net`、`lightmes.user.023ent.net`、`register.lightmes.user.023ent.net`  
> 前提：后端已在宝塔 Python 项目中运行，监听 `127.0.0.1:8000`，且 `curl http://127.0.0.1:8000/api/health` 返回正常。

---

## 一、三个站点分别是什么

| 站点 | 域名示例 | 源码目录 | build 产物 | 谁用 |
|------|----------|----------|------------|------|
| **管理端** | `admin.lightmes.user.023ent.net` | `frontend-admin-pro` | `frontend-admin-pro/dist` | 厂长、平台运营商 |
| **客户端 H5** | `lightmes.user.023ent.net` | `frontend-h5` | `frontend-h5/dist` | 员工扫码报工、客户下单 |
| **注册端** | `register.lightmes.user.023ent.net` | `frontend-portal` | `frontend-portal/dist` | 新企业自助注册（SaaS 路线 B 才需要） |

**核心原则（三个站都一样）：**

1. 站点根目录 = 对应工程的 **`dist`** 目录（纯静态，不要 PHP）
2. **伪静态**：SPA 回退 `index.html`
3. **只反代 `/api/`** 到 `127.0.0.1:8000`，**不要把整站 `/` 反代到后端**
4. 改前端代码后必须重新 `npm run build`，再刷新浏览器

## 二、最容易配错的地方（必读）

在宝塔「网站 → 反向代理」里，**错误示例**：

```nginx
# ❌ 错误：整站反代到 8000 → 打开首页就是 {"detail":"Not Found"}
location ^~ / {
    proxy_pass http://127.0.0.1:8000;
}
```

**正确做法**：后端只提供 API，页面由 `dist` 静态文件提供；Nginx 仅反代 `/api/`：

```nginx
# ✅ 正确
location /api/ {
    client_max_body_size 100m;
    proxy_pass http://127.0.0.1:8000/api/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

## 三、一键构建

```bash
# 管理端 + 客户端（路线 A 单厂 / 路线 B SaaS 都要）
cd /www/wwwroot/lightmes/frontend-admin-pro && npm install && npm run build
cd /www/wwwroot/lightmes/frontend-h5 && npm install && npm run build

# 注册端（仅 SaaS 路线 B 需要）

```

## 四、管理端（admin 站）

### 4.1 宝塔建站

| 字段 | 值 |
|------|-----|
| 域名 | `admin.lightmes.user.023ent.net` |
| 根目录 | `/www/wwwroot/lightmes/frontend-admin-pro/dist` |
| PHP | 纯静态（关闭 PHP） |

### 4.2 伪静态

网站 → 设置 → 伪静态，粘贴（完整示例见 `frontend-admin-pro/nginx-admin.example.conf`）：

```nginx
location /assets/ {
    try_files $uri =404;
    expires 1y;
    add_header Cache-Control "public, immutable";
}

location = /index.html {
    add_header Cache-Control "no-cache, no-store, must-revalidate";
    try_files $uri =404;
}

location / {
    try_files $uri $uri/ /index.html;
}
```

### 4.3 API 反代

网站 → 设置 → 配置文件，在 `server { }` 内加入（若已有「整站反代」，先删掉）：

```nginx
location /api/ {
    client_max_body_size 100m;
    proxy_pass http://127.0.0.1:8000/api/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

### 4.4 SSL

网站 → SSL → Let's Encrypt 申请 → 开启「强制 HTTPS」。

### 4.5 访问地址

| 角色 | URL |
|------|-----|
| 工厂管理员 | `https://admin.lightmes.user.023ent.net/login` 或 `/t/DEMO/login` |
| 平台运营商 | `https://admin.lightmes.user.023ent.net/platform/login` |

演示账号（已执行 `demo_data.py`）：租户 `DEMO` / 用户 `admin` / 密码 `admin123`

### 4.6 验证

```bash
curl -s https://admin.lightmes.user.023ent.net/ | head -3          # 应看到 LightMes 管理后台 HTML
curl -s https://admin.lightmes.user.023ent.net/api/health         # 应返回 {"code":200,...}
```

## 五、客户端 H5（员工 / 客户）

### 5.1 宝塔建站

| 字段 | 值 |
|------|-----|
| 域名 | `lightmes.user.023ent.net` |
| 根目录 | `/www/wwwroot/lightmes/frontend-h5/dist` |
| PHP | 纯静态 |

> **注意**：根目录必须指向 `frontend-h5/dist`，不能指到 `/www/wwwroot/lightmes.user.023ent.net` 这类宝塔默认空目录。

### 5.2 伪静态

```nginx
location /assets/ {
    try_files $uri =404;
    expires 1y;
    add_header Cache-Control "public, immutable";
}

location = /index.html {
    add_header Cache-Control "no-cache, no-store, must-revalidate";
    try_files $uri =404;
}

location / {
    try_files $uri $uri/ /index.html;
}
```

### 5.3 API 反代

与管理端相同，只配 `/api/`：

```nginx
location /api/ {
    client_max_body_size 100m;
    proxy_pass http://127.0.0.1:8000/api/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

H5 前端默认请求相对路径 `/api`（无需单独 `.env.production`），只要同域反代即可。

### 5.4 SSL

同上，申请证书并强制 HTTPS。员工手机扫码、相机报工 **强烈建议 HTTPS**。

### 5.5 访问地址

H5 使用 **Hash 路由**，URL 带 `#`：

| 角色 | URL |
|------|-----|
| 员工登录 | `https://lightmes.user.023ent.net/#/t/DEMO/login` |
| 客户门户 | `https://lightmes.user.023ent.net/#/t/DEMO/customer/order`（`/customer` 会自动跳转至此） |
| 扫码报工 | 扫任务二维码直接进入对应页 |

演示账号（租户 `DEMO`）：

| 角色 | 用户名 | 密码 |
|------|--------|------|
| 员工 | li | 123456 |
| 客户 | customer1 | 123456 |

### 5.6 验证

```bash
curl -s https://lightmes.user.023ent.net/ | head -3
curl -s https://lightmes.user.023ent.net/api/health
```

浏览器打开 `https://lightmes.user.023ent.net/#/t/DEMO/login`，应出现 Vant 风格登录页，而不是 JSON 或「站点创建成功」。

## 六、注册端（portal 站，SaaS 路线 B）

单厂自用（路线 A）**不需要**注册站；只在平台开启「售卖租户」、允许客户自助注册时才部署。

### 6.1 方式一：独立子域名（推荐）

#### 宝塔建站

| 字段 | 值 |
|------|-----|
| 域名 | `register.lightmes.user.023ent.net` |
| 根目录 | `/www/wwwroot/lightmes/frontend-portal/dist` |
| PHP | 纯静态 |

#### 构建

```bash
cd /www/wwwroot/lightmes/frontend-portal
npm install
npm run build
```

可选：构建前复制环境变量，注册成功后跳转管理端用：

```bash
cp .env.production.example .env.production
# 编辑 .env.production：
# VITE_ADMIN_BASE_URL=https://admin.lightmes.user.023ent.net
npm run build
```

#### 伪静态 + API 反代

与管理端、H5 完全相同：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}

location /api/ {
    client_max_body_size 100m;
    proxy_pass http://127.0.0.1:8000/api/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

#### 访问地址

`https://register.lightmes.user.023ent.net/register/`

#### 平台侧开关

1. 登录 `https://admin.lightmes.user.023ent.net/platform/login`
2. **系统设置** → 开启「售卖租户（SaaS 收费模式）」
3. 未开启时，注册页会提示「暂未开放自助注册」

---

### 6.2 方式二：挂在 admin 同域 `/register/`（不新增域名）

适合不想单独申请 `register` 子域名的情况。

#### 构建（必须带 base）

```bash
cd /www/wwwroot/lightmes/frontend-portal
VITE_BASE=/register/ npm run build
```

#### 在 admin 站 Nginx 配置中，在 `location /` 之前增加

```nginx
location ^~ /register/assets/ {
    alias /www/wwwroot/lightmes/frontend-portal/dist/assets/;
}

location ^~ /register/ {
    alias /www/wwwroot/lightmes/frontend-portal/dist/;
    try_files $uri $uri/ /register/index.html;
}
```

admin 站已有的 `/api/` 反代 **不用重复配**，注册页共用同一套 API。

#### 访问地址

`https://admin.lightmes.user.023ent.net/register/`

## 九、常见问题

> **后端启动失败**（`error.log` 里 SQLAlchemy / 8000 端口 / `.env Backend: command not found`）→ 见 **[宝塔部署常见错误排查.md](./宝塔部署常见错误排查.md) 十一～十三**。

### 1. 首页 `{"detail":"Not Found"}`

整站被反代到了 8000。删掉 `location ^~ / { proxy_pass ... }`，只保留 `/api/` 反代。

### 2. 白屏 / 刷新 404

没配伪静态 `try_files`，或根目录不是 `dist`。

### 3. 登录接口 404 / 502

- 检查 `/api/` 反代是否指向 `127.0.0.1:8000`
- 宝塔 Python 项目是否运行中
- 改过后端代码是否在面板里 **重启** 了 Python 项目

### 4. HTTPS 显示「恭喜，站点创建成功」

该域名 **还没申请 SSL**，请求落到了 IP 默认页。在对应站点申请 Let's Encrypt 并强制 HTTPS。

### 5. 注册页「暂未开放自助注册」

平台后台未开启「售卖租户」，或 `POST /api/tenants/register` 被关闭（路线 A 单厂默认关闭，属正常）。

### 6. 更新前端后页面没变化

重新 `npm run build`，确认宝塔根目录指向的 `dist` 已更新，浏览器强刷或清缓存。

## 十一、完整 Nginx 参考片段（单站）

以下为 **H5 客户端** 完整 `server` 块参考，admin / register 只需改 `server_name` 和 `root`：

```nginx
server {
    listen 80;
    listen 443 ssl http2;
    server_name lightmes.user.023ent.net;

    root /www/wwwroot/lightmes/frontend-h5/dist;
    index index.html;

    # SSL 证书路径以宝塔实际为准
    # ssl_certificate     /www/server/panel/vhost/cert/lightmes.user.023ent.net/fullchain.pem;
    # ssl_certificate_key /www/server/panel/vhost/cert/lightmes.user.023ent.net/privkey.pem;

    location /api/ {
        client_max_body_size 100m;
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /assets/ {
        try_files $uri =404;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    location = /index.html {
        add_header Cache-Control "no-cache, no-store, must-revalidate";
        try_files $uri =404;
    }

    location / {
        try_files $uri $uri/ /index.html;
    }

    access_log /www/wwwlogs/lightmes.user.023ent.net.log;
    error_log  /www/wwwlogs/lightmes.user.023ent.net.error.log;
}
```
