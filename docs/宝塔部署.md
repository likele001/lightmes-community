# LightMes 宝塔面板部署指南

> 适用场景：后端 FastAPI（Python项目）+ 前端 Vue 3 SPA（静态站点）
> 所有服务运行在同一台服务器，**生产推荐裸机/宝塔部署**（无 Docker）。
>
> **本地开发 / 演示环境**可选用 Docker：见 **[Docker部署指南.md](./Docker部署指南.md)**（dev 热重载 + staging 三端静态页）。
>
> **请先阅读 [零、选择部署模式](#零选择部署模式必读)**，再按对应路线操作，避免单厂与 SaaS 步骤混在一起。
>
> **想用一个域名访问管理端 + H5 + 注册 + API？** 见 **[单域名部署方案.md](./单域名部署方案.md)**（推荐新部署采用）。

## 零、选择部署模式（必读）

系统支持两种部署方式，**共用同一套后端**，差别在前端要建几个站、要不要平台总控台。


| 对比项              | 路线 A：单厂私有化                         | 路线 B：SaaS 商业化                   |
| ---------------- | ---------------------------------- | ------------------------------- |
| 典型场景             | 只给自己或一家客户部署一套                      | 对外卖账号，多家工厂各自使用                  |
| 要 build 的前端      | **2 个**：admin-pro、h5               | **3 个**：admin-pro、h5、**portal** |
| 平台运营 `/platform` | **可不使用**                           | **必须使用**                        |
| 注册页 `/register`  | **不部署**                            | **需部署** portal                  |
| 新工厂怎么开户          | 演示数据 `DEMO`，或平台里手工建 1 个租户（可选）      | 平台手工建 **或** 客户自助注册 + 支付         |
| 是否 `npm run dev` | 宝塔上线 **只用 `npm run build`**，不要 dev | 同上                              |


### 路线 A 清单（单厂）

完成 **[一～十](#一环境检查)** 公共步骤后，即可使用：


| 角色    | 生产环境访问地址（示例）                                              |
| ----- | --------------------------------------------------------- |
| 工厂管理员 | `https://admin.你的域名/login`（租户编码填 `DEMO`）或 `/t/DEMO/login` |
| 员工报工  | `https://h5.你的域名/#/t/DEMO/login`                          |
| 客户下单  | 同上 H5，`#/t/DEMO/customer/order`                           |


**路线 A 不需要做：** 部署 `frontend-portal`、配置虎皮椒、开启「售卖租户」。  
详细说明见 **[路线 A：单厂私有化](#路线-a单厂私有化)**。

### 路线 B 清单（SaaS）

完成 **[一～十](#一环境检查)** 后，**额外**完成 **[路线 B：SaaS 商业化](#路线-bsaas-商业化)** 全部步骤。


| 角色         | 生产环境访问地址（示例）                                       |
| ---------- | -------------------------------------------------- |
| **你（运营商）** | `https://admin.你的域名/platform/login`                |
| 平台里「租户管理」  | `https://admin.你的域名/platform/tenants`              |
| 新企业自助注册    | `https://register.你的域名/register/`（portal 独立站点，见下表） |
| 某厂管理员      | `https://admin.你的域名/t/厂编码/login`                   |
| 该厂员工/客户    | `https://h5.你的域名/#/t/厂编码/login`                    |

## 速查：`npm run build` 要跑几个目录？（最易乱，先看这里）

> **宝塔上线只用 `npm run build`，不要用 `npm run dev`。**  
> **后端 `backend` 不用 npm**，用 Python + 宝塔「Python 项目」启动。

### 一共几个前端工程？


| 工程目录                 | 路线 A 单厂     | 路线 B SaaS   | 说明                              |
| -------------------- | ----------- | ----------- | ------------------------------- |
| `frontend-admin-pro` | **要 build** | **要 build** | 厂长管理后台 + 平台运营 `/platform`       |
| `frontend-h5`        | **要 build** | **要 build** | 员工报工、客户下单（手机端）                  |
| `frontend-portal`    | 不要          | **要 build** | 仅「新企业自助注册」页                     |

## 二、后端部署（Python 项目）

### 2.1 配置环境变量

```bash
cd /www/wwwroot/lightmes/backend
cp .env.example .env
```

编辑 `.env`，修改以下关键项：

> **`.env` 写法**：带空格的值必须加引号，例如 `APP_NAME="LightMes Backend"`。若写成 `APP_NAME=LightMes Backend`，宝塔启动脚本 `source .env` 时会报 `Backend: command not found`（不影响 pydantic 读文件，但日志会有噪音）。

```ini
# MySQL 连接串
DB_URL=mysql+pymysql://lightmes:你的密码@127.0.0.1:3306/lightmes?charset=utf8mb4

# JWT 密钥（必须修改）
JWT_SECRET=换成随机字符串

# 本地存储根目录
STORAGE_LOCAL_ROOT=./data/storage
```

> ⚠️ 不要把数据库密码写在宝塔「启动命令」里，只放在 `.env` 文件中。

### 2.2 安装依赖与迁移

> **宝塔生产环境必须用 Python 项目虚拟环境的全路径**。不要直接敲 `alembic` / `pip` / 系统 `python3`；也不要假设项目里的 `backend/.venv` 就是线上环境。

#### 本机标准路径（LightMes 宝塔 Python 项目）

```bash
export VENV=/www/server/pyporject_evn/lightmes   # 宝塔 → Python 项目 → lightmes-backend → 环境路径
export ALEMBIC=$VENV/bin/alembic
export PIP=$VENV/bin/pip
export PYTHON=$VENV/bin/python
```

| 说明 | 内容 |
|------|------|
| 标准值 | `/www/server/pyporject_evn/lightmes`（宝塔目录名拼写为 `pyporject_evn`） |
| 如何确认 | 面板 **环境路径** 与上表一致；执行 `$PYTHON -c "import sqlalchemy; print('ok')"` 应输出 `ok` |
| `backend/.venv` | **本地遗留目录，宝塔不会自动使用**。若 `python3` 与 `lib/python3.11` 版本不一致，会报 `No module named 'sqlalchemy'`——请以面板路径为准，或删除后重建 |

#### 安装依赖与迁移

```bash
cd /www/wwwroot/lightmes/backend

$PIP install -r requirements.txt
```

### 4.3 SPA History 模式修复

宝塔 → 网站 → 设置 → 伪静态，添加（**顺序重要**：`/assets/` 须在 `/` 之前，完整示例见 `frontend-admin-pro/nginx-admin.example.conf`）：

```nginx
# 静态 chunk 找不到时返回 404，勿 fallback 到 index.html
location /assets/ {
  try_files $uri =404;
  expires 1y;
  add_header Cache-Control "public, immutable";
}

location = /index.html {
  add_header Cache-Control "no-cache, no-store, must-revalidate";
  try_files $uri =404;
}

location / {
  try_files $uri $uri/ /index.html;
}
```

> **说明**：Vite 每次 `npm run build` 会给 JS 文件名加新 hash。若用户浏览器仍开着旧页面，点菜单会去拉已删除的旧 chunk；若 Nginx 把 `/assets/xxx.js` 404 回退成 `index.html`，Console 会报 `MIME type text/html`，菜单点击无反应。**前端已内置检测到该错误时自动刷新**；运维侧仍建议按上表配置 `/assets/` 与 `index.html` 缓存策略。

### 4.4 API 反向代理（推荐）

在同一个站点的 Nginx 配置中，将 `/api` 指向后端：

```nginx
location /api/ {
  client_max_body_size 100m;
  proxy_pass http://127.0.0.1:8000/api/;
  proxy_set_header Host $host;
  proxy_set_header X-Real-IP $remote_addr;
  proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
}
```

这样前端请求 `/api/xxx` 时无需处理跨域。

## 九、日常维护

### 9.1 数据库升级（更新代码后必须执行）

数据库升级用 **Alembic**，命令格式固定为：

```bash
export VENV=/www/server/pyporject_evn/lightmes   # 宝塔环境路径，见 二、2.2
cd /www/wwwroot/lightmes/backend

# ── 第一步：查看当前版本 ──
$VENV/bin/alembic current
# 输出示例：0054_exec_dashboard_metrics (head)  ← 表示当前在这里

# ── 第二步：执行升级 ──
$VENV/bin/alembic upgrade head
# 输出示例：Running upgrade 0054_... -> 0055_rag_feedback -> 0056_predict_models, ...
# 如果已经是最新版本，输出为空，表示无需升级

# ── 第三步：确认升级结果 ──
$VENV/bin/alembic current
# 应输出：0056_predict_models (head)
```

> **每次更新代码后，必须执行上述三步。** 新迁移文件不会自动执行，必须手动 `upgrade head`。
> **宝塔重启** Python 项目（uvicorn）后新接口才会生效。

**常见报错处理**

| 报错 | 原因 | 解决 |
|------|------|------|
| `FAILED: Target database is up to date` | 已是最新，无需操作 | 正常，不用管 |
| `Table already exists` | 旧版手动建过表，alembic_version 为空 | 执行 `$VENV/bin/alembic stamp <已执行到的版本号>` 跳过 |
| `No module named 'alembic'` | 用了系统 python3 而不是虚拟环境 | 确认 `$VENV/bin/alembic` 路径存在 |
| `Can't locate revision identified by ...` | 本地 alembic_version 指向了仓库里不存在的版本 | 手动执行 `UPDATE alembic_version SET version_num = '0054_exec_dashboard_metrics'`（改成仓库里有的版本） |

## 十、常见问题

> **503 / PIL / Redis / 端口占用 / Python3.14+SQLAlchemy / 静态 404** 等详细排错见专文：  
> **[宝塔部署常见错误排查.md](./宝塔部署常见错误排查.md)**（含 `error.log` 路径、SQLAlchemy 升级、8008 端口冲突）

### Python 项目启动失败

查看宝塔 Python 项目日志，常见原因：

- 依赖未装齐 → 确认已执行 `$VENV/bin/pip install -r requirements.txt`（`VENV` 见 **二、2.2**）
- **Python 3.14 + SQLAlchemy 2.0.0 启动崩溃**（`error.log` 含 `SQLCoreOperations` / `__static_attributes__`）→ 见 **[宝塔部署常见错误排查.md](./宝塔部署常见错误排查.md) 十一**
- **`ModuleNotFoundError: No module named 'PIL'`** → `$VENV/bin/pip install Pillow` 后重启（验证码必需，见错误排查文档 **三**）
- 浏览器 **503** 且 `error.log` 有 PIL/端口错误 → 见 **[宝塔部署常见错误排查.md](./宝塔部署常见错误排查.md) 二、三、六**
- `No module named 'sqlalchemy'` 且你在用 `backend/.venv` → 用错了环境；改走宝塔 **环境路径** `/www/server/pyporject_evn/lightmes`
- `.env` 未加载 → 确认启动配置中选择了「从文件加载」
- MySQL 连接不上 → 检查 DB_URL 和 MySQL 权限
- 端口被占用 → 检查 8000 端口是否被其他服务占用

### MySQL 连接被拒绝

```sql
-- 给 lightmes 用户授权
GRANT ALL PRIVILEGES ON lightmes.* TO 'lightmes'@'localhost' IDENTIFIED BY '你的密码';
FLUSH PRIVILEGES;
```

### 前端页面白屏 / 404

- 确认根目录指向 dist 目录（不是项目根目录）
- 确认已配置 `try_files $uri $uri/ /index.html;`
- **`/assets/` 须单独配置 `try_files $uri =404;`**，勿让缺失的 JS 回退成 `index.html`（见 4.3 节）

### 点菜单没反应 / Console 报 Failed to fetch dynamically imported module

**现象**：地址栏不变；Console 出现 `MIME type of "text/html"`、`Failed to fetch dynamically imported module: .../assets/HomePage-xxx.js`。

**原因**：页面开着时又执行了 `npm run build` 部署，浏览器仍引用旧 hash 的 chunk，服务器上该文件已不存在；Nginx 把 `/assets/*.js` 404 误返回了 `index.html`。

**处理**：

1. **立即**：F5 或 Ctrl+F5 刷新（仍保持登录）
2. **Nginx**：按 4.3 节为 `/assets/` 配置 `try_files $uri =404;`，`index.html` 设 `no-cache`
3. **部署**：每次 build 后整包替换 `dist/`，不要只覆盖部分文件
4. **前端**：管理端/H5 已内置 chunk 加载失败时自动刷新一次（需重新 build 前端后生效）

### 前端接口 401

- 确认用户已登录
- 确认 API 反代配置正确
- 检查后端日志确认请求是否到达

### 报工/审核上传视频失败

后端 `.env` 默认单文件 **100MB**（`FILE_MAX_UPLOAD_SIZE`），并支持 MP4、MOV（iPhone）、WebM 等。

若仍失败，检查 Nginx **admin 站**与 **H5 站** 反代配置，在 `location /api/` 内增加（或调大）：

```nginx
client_max_body_size 100m;
```

修改后重载 Nginx，并 **重启** Python 项目使 `.env` 生效。

### 报工视频必须现场拍摄

管理端 **系统设置 → 报工拍摄设置** 可配置：


| 项      | 默认   | 说明           |
| ------ | ---- | ------------ |
| 单段视频最长 | 15 秒 | 到点自动结束，避免长视频 |
| 单段视频上限 | 8 MB | 每段单独上传       |
| 最多视频段数 | 3 段  | 多件证据分多次拍     |
| 最多照片张数 | 5 张  | 员工报工现场拍照     |


H5/审核页使用 **摄像头录制**，不支持从相册选大文件；后端 `purpose=report_media` 会按租户配置再校验大小。

**手机 Edge 仍提示无法摄像**：多为站点使用 **HTTP**（非 HTTPS），浏览器禁止页内 `getUserMedia`。已改为自动 **调起系统相机录像**；仍建议宝塔为 `lightmes`、`admin` 站点开启 **SSL**，体验更好。

`backend/.env` 配置（修改后需重启 Python 项目）：

```env
ACCESS_TOKEN_EXPIRE_MINUTES=480          # 未勾选记住登录：8 小时
```

| 选项       | 后端 token 有效期 | 浏览器行为                                 |
| -------- | ------------ | ------------------------------------- |
| 不勾选      | 8 小时         | token 存 **sessionStorage**，关闭浏览器需重新登录 |

### B.4 注册门户 frontend-portal 部署（路线 B 必做）

> **和 admin、H5 一样**：必须是 **伪静态（SPA）+ `/api/` 反代**，否则注册页白屏或接口 404。

#### B.4.1 构建

**方式 1：独立子域名（推荐，根目录即 dist）**

```bash
cd /www/wwwroot/lightmes/frontend-portal
npm install
npm run build
```

**方式 2：挂在 admin 同域 `/register/` 下**

```bash
cd /www/wwwroot/lightmes/frontend-portal
VITE_BASE=/register/ npm run build
```

产物：`/www/wwwroot/lightmes/frontend-portal/dist`

#### B.4.2 方式 1：独立站点（推荐，配置最简单）

宝塔 → 网站 → 添加站点：


| 字段  | 值                                                     |
| --- | ----------------------------------------------------- |
| 域名  | `register.你的域名`（例：`register.lightmes.user.zuiger.cn`） |
| 根目录 | `/www/wwwroot/lightmes/frontend-portal/dist`          |
| PHP | 纯静态                                                   |


**伪静态**（网站 → 设置 → 伪静态，整段粘贴）：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

**API 反代**（网站 → 设置 → 配置文件，在 `server { }` 内、`#SSL` 等注释前加入）：

```nginx
location /api/ {
    proxy_pass http://127.0.0.1:8000/api/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

访问：`https://register.你的域名/register/`（路由默认会跳到 `/register`）

管理端登录页的「注册新企业」若链到同域，需把链接改成该完整 URL；或采用下面 **方式 2** 与 admin 同域。

#### B.4.3 方式 2：挂在 admin 站点同域（不新增域名）

1. 用 `**VITE_BASE=/register/ npm run build`** 构建（见 B.4.1）
2. 在 **admin 站点** 的 Nginx 配置里，在已有 `location /` **之前** 增加：

```nginx
# 注册门户 SPA（目录指向 portal 的 dist）
location ^~ /register/ {
    alias /www/wwwroot/lightmes/frontend-portal/dist/;
    try_files $uri $uri/ /register/index.html;
}

location ^~ /register/assets/ {
    alias /www/wwwroot/lightmes/frontend-portal/dist/assets/;
}
```

1. admin 站点 **已有** 的伪静态 + `/api/` 反代 **保留不动**（注册页也走同一套 `/api/` 反代）。

访问：`https://admin.你的域名/register/`

### B.8 迁移版本与表已存在（路线 A/B 均可能遇到）

库表已存在但 `alembic_version` 为空时，勿直接 `upgrade head`（会报 Table already exists）。

```bash
export VENV=/www/server/pyporject_evn/lightmes   # 以面板环境路径为准
export ALEMBIC=$VENV/bin/alembic
cd /www/wwwroot/lightmes/backend

$ALEMBIC current          # 查看当前版本
# 若表已齐全但版本落后，可标记到最新（确认 SaaS 表、code_sequences 等已存在后）：
$ALEMBIC stamp 0027_code_sequences
```

SaaS 升级 halfway 报 `Duplicate column` 时：说明列已加过，执行 `$ALEMBIC stamp 0027_code_sequences`（或当时仓库 `alembic/versions/` 下最新 revision 名）即可。

> **说明**：`0027_code_sequences` 为统一业务编号序号表；若你仓库更新后出现 `0028_xxx`，以 `alembic history` 最后一项为准。
