# LightMes 飞书消息推送部署指南

> 适用：LightMes 已按 [宝塔部署.md](./宝塔部署.md) 完成后端 + Admin + H5 基础部署。  
> 本文只讲 **飞书（Lark）第二通道**：派工、报工审核、工资、预警等消息的飞书推送与 OAuth 绑定。  
> 站内通知（铃铛）仍保留，与飞书并行。

---

## 一、功能概览

| 能力 | 说明 |
|------|------|
| 消息卡片 | 默认以 interactive 卡片推送，含「查看详情」跳转 H5/Admin |
| 卡片审核 | 报工待审卡片支持「初审通过 / 驳回」（需事件订阅 + 用户已绑定 open_id） |
| OAuth 绑定 | Admin / H5 个人中心一键绑定飞书账号 |
| 手机号匹配 | Admin 按手机号批量匹配飞书 open_id |
| 群组推送 | 管理群、生产群、全厂群；部门可映射专属群 |
| 静默时段 | 22:00–07:00 等时段延迟发送，Celery Beat 每 5 分钟 flush |
| 推送日志 | Admin 可查看 pending / deferred / success / failed 并重试 |

### 默认推送路由

| 事件 | 默认目标 |
|------|----------|
| 派工 `dispatch.assigned` | 被派工员工 |
| 报工提交 `report.submitted` | 部门班组长、车间负责人 |
| 报工初审/终审/驳回 | 报工员工 |
| 工资条催签/重置/拒签 | 员工；拒签额外通知老板 + 管理群 |
| 客户下单 | 订单管理权限 + 管理群 |
| AI/业务预警 `alert` | 按级别升级：部门管理 → 老板 → 管理群 → 全厂群 |
| 每日简报 `brief.daily` | 老板 + 管理群 + 全厂群 |
| 生产自动化失败 | 计划管理权限 + 管理群 |

规则可在 **系统管理 → 飞书消息推送 → 推送规则** 中调整。

## 二、前置条件

1. **LightMes 后端可公网 HTTPS 访问**（飞书回调必须 HTTPS）
2. **MySQL、Redis 已运行**（Redis 供 Celery 使用）
3. **Celery Worker + Beat 已启动**（若已按宝塔部署第八节启动过则通常已有；**上线飞书功能后须重启一次**，见 [八、Celery](#八celery必开)）
4. 已在 [飞书开放平台](https://open.feishu.cn/app) 创建 **企业自建应用**
5. Admin、H5 前端已重新构建（含飞书配置页与 H5 绑定入口）

## 四、域名与 Nginx

飞书需要 **两个对外 URL**（可与 Admin/H5 同域或分域）：

| 用途 | 路径 | 示例 |
|------|------|------|
| 事件订阅 / 卡片回调 | `POST {API}/api/feishu/events` | `https://api.example.com/api/feishu/events` |
| OAuth 绑定回调 | `GET {API}/api/feishu/oauth/callback` | `https://api.example.com/api/feishu/oauth/callback` |

**API 外网地址** = 浏览器能访问到 `/api/health` 的根 URL（不含 `/api` 后缀）。

### 多域名（常见）

- Admin：`https://admin.example.com` → 静态 + `location /api/` 反代 `127.0.0.1:8000`
- H5：`https://h5.example.com` → 同上
- API 外网地址填：`https://admin.example.com`（与 Admin 同域反代 API 时）

### 单域名

若 Admin、H5、API 共用一个域名（如 `https://mes.example.com`），在后台分别填写：

- **API 外网地址**：`https://mes.example.com`
- **Admin 外网地址**：`https://mes.example.com`（或 Admin 子路径根）
- **H5 外网地址**：`https://mes.example.com/h5`（按实际 H5 部署路径）

> 卡片「查看详情」跳转依赖上述三个地址，填错会导致按钮打开 404。

Nginx 需保证 **`/api/feishu/events`** 可被飞书服务器 POST（无登录拦截、无 WAF 误拦 JSON body）。

## 五、后端环境变量（可选兜底）

租户级配置优先存 **Admin → 飞书消息推送**；以下 `.env` 项作全局默认：

```env
# API 外网根地址（生成回调 URL；Admin 未填「API 外网地址」时使用）
PUBLIC_BASE_URL=https://admin.example.com

# H5 外网根地址（卡片跳转员工端；Admin 未填时使用）
H5_PUBLIC_BASE_URL=https://h5.example.com

# 可选：不在 Admin 填 App 凭证时的环境变量兜底
FEISHU_APP_ID=cli_xxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxx
```

修改 `.env` 后重启 Python 项目。

## 六、飞书开放平台配置

登录 [飞书开放平台](https://open.feishu.cn/app) → 选择应用 → 按下列步骤操作。

### 6.1 凭证

**凭证与基础信息** 页记录：

- **App ID**（`cli_xxx`）
- **App Secret**

稍后填入 LightMes Admin。

### 6.2 权限（Scopes）

在 **权限管理** 中申请并 **发布版本**（未发布权限不生效）。建议至少包含：

| 权限 | 用途 |
|------|------|
| 获取 tenant_access_token | 发消息、拉群、查用户 |
| 以应用身份发消息 | 推送个人/群消息 |
| 获取群组信息 / 读取群列表 | Admin「拉取机器人所在群」 |
| 获取用户 userid / 通过手机号查用户 | OAuth、手机号批量匹配 |
| 获取部门组织架构 | Admin「拉取飞书部门」 |
| 获取用户基本信息 | OAuth 绑定 |

卡片审核还需在 **事件订阅** 中订阅 **`card.action.trigger`**（见 6.4）。

> 权限名称以飞书控制台为准，搜索「消息」「通讯录」「群组」「身份验证」相关项即可。

### 6.3 机器人能力

**应用能力 → 机器人**：

1. 启用机器人
2. 将机器人 **加入** 需要接收通知的飞书群（生产群、管理群、全厂群等）
3. 群内 @机器人 发一条测试，确认机器人在线

### 6.4 事件订阅

**事件订阅** 页：

| 配置项 | 值 |
|--------|-----|
| 请求地址 URL | `{API外网}/api/feishu/events` |
| Encrypt Key | 自定义字符串，**与 Admin 中 Encrypt Key 一致** |
| Verification Token | 自定义字符串，**与 Admin 中 Verification Token 一致** |

订阅事件：

- **`card.action.trigger`**（卡片按钮回调，报工初审/驳回必需）

保存后飞书会发 **URL 验证** 请求；LightMes 会自动返回 `challenge`，验证通过即可。

> 若启用 Encrypt Key，请求体为加密 JSON，后端会用 Admin 配置的 key 解密。

### 6.5 OAuth 重定向

**安全设置 → 重定向 URL** 添加：

```text
{API外网}/api/feishu/oauth/callback
```

示例：

```text
https://admin.example.com/api/feishu/oauth/callback
```

**网页应用** 或 **登录** 相关能力需开启（用于员工 OAuth 绑定 open_id）。

### 6.6 发布应用

**版本管理与发布** → 创建版本 → 提交审核 / 企业内发布。  
未发布的应用权限与事件订阅对正式环境可能无效。

## 七、LightMes Admin 配置

路径：**系统管理 → 飞书消息推送**（需 `setting.manage` 权限）。

### 7.1 基础连接

| 字段 | 说明 |
|------|------|
| 启用飞书推送 | 总开关 |
| App ID / App Secret | 飞书应用凭证 |
| 企业 Tenant Key | 可选；多租户 SaaS 时用于事件解密识别租户 |
| Encrypt Key / Verification Token | 与飞书事件订阅一致 |
| H5 / Admin / API 外网地址 | 卡片跳转与回调 URL 生成 |
| 消息形式 | 推荐 **消息卡片** |
| 卡片审核按钮 | 开启后报工待审卡片显示初审/驳回 |
| 静默时段 | 可选；开启后非紧急消息延迟到时段外发送 |

保存后页面会显示 **事件回调地址**、**OAuth 绑定回调**，应与飞书控制台完全一致。

点击 **测试连接**，应提示 token 获取成功。

### 7.2 群组管理

1. 点击 **拉取机器人所在群**
2. 为「生产群 / 管理群 / 全厂群」填写 `chat_id`（`oc_xxx`）
3. 可选填 **Webhook URL**（群自定义机器人 webhook，作为 chat_id 发送的补充通道）

### 7.3 推送规则

按事件勾选 **推送目标**（员工、班组长、老板、各群等）。默认规则见本文第一节。

### 7.4 人员绑定

员工飞书 **open_id** 是推送到个人的必要条件。三种方式：

| 方式 | 操作 |
|------|------|
| OAuth（推荐） | Admin「绑定我的飞书」或 H5 **个人中心 → 绑定飞书** |
| 手动 | Admin 人员绑定表直接填 `open_id` |
| 手机号批量匹配 | 员工手机号与飞书一致时，点 **按手机号批量匹配** |

### 7.5 部门映射

1. **拉取飞书部门**
2. 为 LightMes 部门选择 `open_department_id`
3. 可选 **关联通知群**（对应群组管理中的群标识），用于 `group:dept_auto` 目标

### 7.6 测试发送

**测试发送** Tab：

- `receive_id_type = open_id`，`receive_id = ou_xxx` → 测个人
- `receive_id_type = chat_id`，`receive_id = oc_xxx` → 测群
- `receive_id_type = webhook`，填 webhook 完整 URL → 测群 webhook

### 7.7 卡片预览

**卡片预览** Tab 可查看报工待审等事件的卡片 JSON，确认审核按钮 `value` 含 `action / biz_type / biz_id / tenant_id`。

## 八、Celery（必开）

飞书消息通过 Celery **异步发送**；静默队列依赖 Beat 定时任务 **`feishu.flush_deferred`**（每 5 分钟 flush 一次 `deferred` 记录）。

> **若你已按 [宝塔部署.md](./宝塔部署.md) 第八节启动过 Celery，通常 Worker + Beat 已在运行，不必从零安装。**  
> 但 **上线/更新飞书代码后必须重启 Celery**，否则 Beat 不会加载新的 `feishu.flush_deferred` 调度，Worker 也可能未注册 `feishu.send_message` 任务。

### 8.1 我有没有 Celery？

在服务器上执行：

```bash
ps aux | grep 'celery -A app.celery_app'
```

若能看到类似下面 **两行**（或多条 worker 子进程 + 一条 beat），说明已在跑：

```text
... celery -A app.celery_app worker -l info
... celery -A app.celery_app beat -l info
```

LightMes 项目自带脚本（依赖 **Redis 已启动**）：

| 操作 | 命令 |
|------|------|
| 启动 | `bash /www/wwwroot/lightmes/scripts/start-celery.sh` |
| 停止 | `bash /www/wwwroot/lightmes/scripts/stop-celery.sh` |

日志目录（默认）：

```text
/tmp/lightmes-celery/worker.log
/tmp/lightmes-celery/beat.log
/tmp/lightmes-celery/worker.pid
/tmp/lightmes-celery/beat.pid
```

手动启动（与脚本等价）：

```bash
export VENV=/www/server/pyporject_evn/lightmes
cd /www/wwwroot/lightmes/backend
$VENV/bin/celery -A app.celery_app worker -l info &
$VENV/bin/celery -A app.celery_app beat -l info &
```

### 8.2 飞书相关任务

| 任务名 | 触发方式 | 说明 |
|--------|----------|------|
| `feishu.send_message` | 业务事件入队后立即 | 发送单条推送（文本/卡片） |
| `feishu.flush_deferred` | Beat 每 **5 分钟** | 静默时段结束后批量发出 `deferred` 消息 |

Beat 调度定义在 `backend/app/celery_app.py` 的 `feishu-flush-deferred` 项。

### 8.3 更新飞书代码后：必须重启 Celery

Celery **Beat 启动时加载调度表，不会热更新**。若 Beat 在飞书功能上线之前就已启动，日志里只有 `ai.daily_brief.send`、`crm.public_pool.auto_recycle` 等旧任务，**不会出现 `feishu-flush-deferred`**，静默队列将一直积压。

**推荐每次拉取含飞书的代码并完成迁移后执行：**

```bash
bash /www/wwwroot/lightmes/scripts/stop-celery.sh
bash /www/wwwroot/lightmes/scripts/start-celery.sh
```

验证 Beat 已注册飞书任务：

```bash
grep feishu /tmp/lightmes-celery/beat.log
# 期望看到类似：
# Scheduler: Sending due task feishu-flush-deferred (feishu.flush_deferred)
```

验证 Worker 能执行发送（触发一次测试推送后）：

```bash
grep feishu /tmp/lightmes-celery/worker.log
# 期望看到 feishu.send_message 相关 Received/Task succeeded
```

> **说明：** 若 Admin 尚未启用飞书、或未发生派工/报工等业务，日志里暂时没有 `feishu` 字样也正常；重启后只要 `beat.log` 在整点/每 5 分钟出现 `feishu-flush-deferred` 即表示调度生效。

### 8.4 故障现象

| 现象 | 常见原因 |
|------|----------|
| 推送日志长期 `pending` | Worker 未启动，或 Redis 不可达 |
| 推送日志长期 `deferred` 不发出 | Beat 未启动，或 **未重启 Beat**（见 8.3） |
| `beat.log` 无 `feishu-flush-deferred` | Beat 进程过旧，需 stop + start |

Redis 未启动时 Celery 无法工作。详见 [宝塔部署常见错误排查.md](./宝塔部署常见错误排查.md)。

## 十二、常见问题

### 1. 测试连接失败 / token 获取失败

- 检查 App ID、App Secret 是否正确
- 飞书应用是否已 **发布** 且权限已生效
- 服务器能否访问 `open.feishu.cn`（防火墙/出站限制）

### 2. 事件订阅 URL 验证失败

- 确认 Nginx 将 `/api/feishu/events` 反代到后端，且 **允许 POST**
- `PUBLIC_BASE_URL` 或 Admin「API 外网地址」必须与飞书填写的域名一致（含 `https`）
- 后端 Python 项目已重启、迁移已执行

### 3. 个人收不到消息，群 Webhook 可以

**个人 open_id 推送 = 飞书「与 lightmes 机器人的单聊」，不是群消息，也不是「应用消息」Tab。**

LightMes 后台「飞书消息推送」页顶部的 **个人推送配置检查** 会自动检测线上版本是否缺少必配项。当前最常见缺项：

| 缺项 | 处理 |
|------|------|
| 未订阅 `im.message.receive_v1` | 开放平台 → 事件与回调 → 事件配置 → 添加「接收消息」→ 回调 URL 填本页「事件回调地址」→ **发布新版本** |
| 未开通 `im:message.p2p_msg:readonly` | 权限管理 → 读取用户发给机器人的单聊消息 → 开通 → 发布 |
| 管理后台未展示应用 | [飞书管理后台](https://feishu.cn/admin) → 应用管理 → lightmes → 全部成员 + **在客户端展示** |
| 员工未激活单聊 | OAuth 绑定后 **必须** 从工作台打开 lightmes，向机器人发「测试」 |

**员工绑定流程（必做）：**

1. H5/Admin 点击「绑定飞书」完成 OAuth
2. 绑定成功页点击 **「打开 LightMes 机器人（必点）」**
3. 在机器人对话里发送任意消息（如「测试」）
4. 之后派工/报工/工资通知会出现在该单聊

可选：LightMes 开启 **「个人消息加急」**（需额外开通 `im:message.urgent`），派工等通知会应用内加急提醒。

服务端若返回 `message_id` 但客户端看不到，99% 是以上配置未完成，不是 LightMes 未发送。

### 4. 卡片按钮点击无反应 / 提示无权限

- 事件订阅是否包含 **`card.action.trigger`**
- Encrypt Key 是否与飞书控制台一致
- 点击者是否已在 LightMes 绑定 **同一 open_id**
- 点击者是否具备报工审核权限（班组长 / `report.audit`）

### 5. 推送日志一直 pending

- **Celery worker 未启动** 或 Redis 不可用
- 查看 worker 日志是否有 `feishu.send_message` 报错

### 6. 推送日志 deferred 不发出

- **Celery beat 未启动**（`feishu.flush_deferred` 每 5 分钟）
- 仍在静默时段内，`scheduled_at` 未到

### 7. OAuth 绑定失败

- 飞书 **重定向 URL** 是否已添加且与 Admin 显示完全一致
- Admin 是否已配置 App ID 与 API 外网地址
- 绑定链接是否过期（state 默认 30 分钟有效）

### 8. 卡片「查看详情」404

- 检查 Admin / H5 **外网地址** 是否与真实部署域名一致
- H5 是否已 `npm run build` 且 Nginx `try_files` 配置正确

### 9. 手机号批量匹配 0 条

- 员工档案手机号是否与飞书账号一致（含国家码格式）
- 飞书是否开通「通过手机号查用户」权限

## 十三、安全说明

1. **App Secret、Encrypt Key** 仅存数据库 / `.env`，勿提交 Git
2. `/api/feishu/events` 为公开接口，依赖飞书 Encrypt Key 与 Verification Token；生产环境 **务必启用加密**
3. OAuth `state` 为短期 JWT，仅用于绑定，不可复用作登录
4. 卡片审核会校验操作人 open_id 与 LightMes 用户权限，勿在飞书侧跳过权限管理
